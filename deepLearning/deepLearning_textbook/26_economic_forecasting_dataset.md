# 26. 경제 시계열 예측용 데이터셋 만들기

## 이번 장의 목표

25장에서는 FRED API로 경제 지표를 가져오는 방법을 배웠다.

이번 장에서는 가져온 경제 지표를 예측 문제용 데이터셋으로 바꾼다.

목표는 다음 한 문장이다.

> 여러 경제 지표를 날짜 기준으로 정리하고, 과거 값을 이용해 다음 달 CPI를 예측하는 데이터셋을 만든다.

이번 장에서는 모델 학습보다 데이터셋 설계가 중요하다.

## 예측 문제 정의

이번 장의 예측 목표는 다음과 같다.

```text
과거 경제 지표를 보고 다음 달 CPI를 예측한다.
```

조금 더 구체적으로 쓰면 다음과 같다.

```text
입력 X = 이번 달과 과거 몇 달의 CPI, 실업률, 금리
정답 y = 다음 달 CPI
```

이것은 회귀 문제다.

정답이 “상승/하락” 같은 종류가 아니라 CPI라는 숫자이기 때문이다.

## target 만들기

경제 시계열 예측에서 가장 중요한 작업 중 하나가 target 만들기다.

예를 들어 현재 행이 2020년 1월이라면, 정답은 2020년 2월 CPI가 될 수 있다.

```text
2020-01의 입력 -> 2020-02 CPI 예측
2020-02의 입력 -> 2020-03 CPI 예측
2020-03의 입력 -> 2020-04 CPI 예측
```

pandas에서는 `shift()`를 사용해 다음 달 값을 현재 행의 정답으로 붙일 수 있다.

```python
df["target_next_cpi"] = df["cpi"].shift(-1)
```

여기서 `shift(-1)`은 한 칸 위로 당긴다는 뜻이다. 즉 다음 시점의 값이 현재 행에 붙는다.

## lag feature란 무엇인가

lag는 지연이라는 뜻이다.

경제 시계열에서는 과거 값을 입력 특성으로 자주 사용한다.

예를 들어 CPI의 1개월 전 값, 2개월 전 값, 3개월 전 값을 만들 수 있다.

```text
cpi_lag_1 = 1개월 전 CPI
cpi_lag_2 = 2개월 전 CPI
cpi_lag_3 = 3개월 전 CPI
```

pandas에서는 `shift(1)`, `shift(2)`, `shift(3)`으로 만든다.

```python
df["cpi_lag_1"] = df["cpi"].shift(1)
```

## 미래 정보 누출 조심

target은 다음 달 CPI이므로 미래 값이다.

하지만 입력 X에는 미래 값을 넣으면 안 된다.

허용되는 정보:

```text
현재 달 정보
과거 달 정보
```

허용되지 않는 정보:

```text
다음 달 CPI
다음 달 실업률
다음 달 금리
```

미래 정보를 입력에 넣으면 모델이 실제 예측 상황보다 쉬운 문제를 풀게 된다.

## 시간 순서 split

경제 시계열에서도 무작위 split은 조심해야 한다.

이번 장에서는 앞쪽 80%를 학습용, 뒤쪽 20%를 검증용으로 사용한다.

```text
과거 80% -> 학습
미래 20% -> 검증
```

이 방식이 실제 예측 상황에 더 가깝다.

## 이번 장의 실습 흐름

```text
저장된 FRED CSV 읽기
-> 날짜 순서 정렬
-> target_next_cpi 생성
-> lag feature 생성
-> 결측치 제거
-> train/validation 시간 순서 split
-> StandardScaler 적용
-> X/y shape 확인
```

## 이번 장의 과제

1. `target_next_cpi`가 무엇을 의미하는지 설명해보자.
2. `shift(-1)`과 `shift(1)`의 차이를 설명해보자.
3. lag feature가 왜 필요한지 설명해보자.
4. 경제 시계열에서 미래 정보 누출이 왜 위험한지 설명해보자.
