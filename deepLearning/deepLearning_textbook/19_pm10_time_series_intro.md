# 19. PM10 시계열 데이터 입문

## 이번 장의 목표

이전 장까지는 이미지 데이터와 표 데이터를 다뤘다.

이번 장부터는 시간 순서가 있는 데이터를 다룬다.

이번 장의 목표는 다음 한 문장이다.

> 서울 PM10 데이터를 읽고, 시간 순서가 있는 데이터가 일반 표 데이터와 어떻게 다른지 이해한다.

PM10은 미세먼지 농도를 나타내는 대표 지표 중 하나다.

이번 프로젝트는 환경/건강 주제와 연결된다.

```text
대기질 데이터
-> 시간 흐름 확인
-> 과거 PM10 패턴으로 다음 시점 PM10 예측
-> 건강/환경 예측 프로젝트로 확장
```

## 사용할 데이터

이번 장에서는 다음 파일을 사용한다.

```text
C:\work\dataset\seoul_pm10.csv
```

데이터 컬럼은 다음과 같다.

```text
date   = 측정 시간

```

인코딩은 글자를 어떤 규칙으로 저장했는지를 뜻한다. 한국어 CSV 파일을 다룰 때는 `utf-8`, `utf-8-sig`, `cp949`, `euc-kr` 중 하나를 맞춰야 하는 경우가 많다.

## 시계열 데이터란 무엇인가

시계열 데이터는 시간 순서가 중요한 데이터다.

예를 들면 다음과 같다.

```text
시간별 미세먼지 농도
일별 주가
월별 소비자물가지수
시간별 자전거 대여량
분 단위 전력 사용량
```

시계열 데이터에서는 행을 무작위로 섞으면 안 되는 경우가 많다.

왜냐하면 과거에서 미래로 이어지는 순서 자체가 정보이기 때문이다.

## 일반 표 데이터와 무엇이 다른가

당뇨병 예측에서는 한 행이 한 사람의 건강 지표였다.

그 행들은 대체로 독립적인 샘플처럼 다룰 수 있었다.

하지만 PM10 데이터에서는 한 행이 특정 시간의 미세먼지 값이다.

```text
2022-01-01 00:00
2022-01-01 01:00
2022-01-01 02:00
...
```

여기서는 이전 시간의 값이 다음 시간의 값과 관련될 수 있다.

즉, 시간 순서가 모델의 입력 정보가 된다.

## 미래 정보를 몰래 보면 안 된다

시계열에서 특히 조심해야 하는 것이 미래 정보 누출이다.

예를 들어 2022년 1월부터 10월까지로 학습하고, 11월 데이터를 검증한다고 하자.

이때 11월 정보를 이용해 1월 데이터를 전처리하면 안 된다.

시계열에서는 다음 원칙을 지켜야 한다.

```text
과거 데이터로 학습
미래 데이터로 검증
```

무작위 train/test split을 쓰면 시간 순서가 섞여 미래 정보가 학습에 들어갈 수 있다.

## 결측치란 무엇인가

실제 센서 데이터에는 빈 값이 있을 수 있다.

PM10 데이터에도 결측치가 있다.

결측치 처리 방법에는 여러 가지가 있다.

```text
삭제하기
앞 시간 값으로 채우기
뒤 시간 값으로 채우기
보간하기
지역 평균으로 채우기
```

처음에는 단순하게 결측치를 제거하거나 앞 시간 값으로 채우는 방법부터 배운다.

## 지역을 하나만 고르는 이유

데이터에는 여러 지역구가 들어 있다.

처음부터 26개 지역을 모두 다루면 복잡해진다.

그래서 입문 단계에서는 한 지역만 선택한다.

예를 들어 `강남구`만 고르면 다음처럼 단순한 시계열이 된다.

```text
강남구의 시간별 PM10
```

한 지역의 흐름을 이해한 뒤, 나중에 여러 지역을 함께 다룰 수 있다.

## 이번 장의 실습 흐름

이번 장의 노트북은 다음 순서로 진행된다.

```text
CSV 읽기
-> 인코딩 확인
-> 날짜 컬럼 datetime 변환
-> 지역 목록 확인
-> 강남구 데이터만 선택
-> 시간 순서 정렬
-> 결측치 확인
-> PM10 시각화
-> 이동평균으로 흐름 보기
```

이번 장에서는 아직 LSTM을 만들지 않는다.

먼저 시계열 데이터가 어떤 모양인지 보는 것이 중요하다.

## 이번 장의 과제

1. 시계열 데이터에서 행을 무작위로 섞으면 왜 문제가 될 수 있는지 설명해보자.
2. PM10 데이터에서 `date`, `area`, `pm10`, `pm2.5`가 각각 무엇을 의미하는지 적어보자.
3. 미래 정보 누출이 무엇인지 자기 말로 설명해보자.
4. 결측치를 처리하는 방법을 세 가지 적어보자.
