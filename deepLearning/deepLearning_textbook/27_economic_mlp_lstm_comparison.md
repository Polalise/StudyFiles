# 27. 경제 시계열 예측: MLP와 LSTM 비교하기

## 이번 장의 목표

26장에서는 다음 달 CPI를 예측하기 위한 경제 시계열 데이터셋을 만들었다.

이번 장에서는 두 가지 딥러닝 모델을 비교한다.

```text
MLP  = lag feature를 입력으로 받는 기본 Dense 모델
LSTM = 시간 window를 입력으로 받는 시계열 모델
```

목표는 다음 한 문장이다.

> 같은 경제 시계열 예측 문제를 MLP 방식과 LSTM 방식으로 풀어보고, MAE/RMSE로 비교한다.

## 왜 MLP와 LSTM을 비교할까

경제 시계열이라고 해서 무조건 LSTM을 써야 하는 것은 아니다.

과거 값을 `lag feature`로 잘 만들어 두면 MLP도 시계열 예측에 사용할 수 있다.

예를 들어 다음 입력을 MLP에 넣을 수 있다.

```text
cpi
cpi_lag_1
cpi_lag_2
cpi_lag_3
unemployment_rate
fed_funds_rate
...
```

이 방식은 시간 순서를 직접 읽는 것은 아니지만, 과거 값들을 특성으로 넣어 시간 정보를 간접적으로 제공한다.

반면 LSTM은 입력을 이런 모양으로 받는다.

```text
(samples, timesteps, features)
```

즉, 과거 몇 개월의 흐름을 순서가 있는 묶음으로 본다.

## MLP 방식

MLP 방식은 26장에서 만든 lag feature 테이블을 그대로 사용한다.

입력 모양은 다음과 같다.

```text
(샘플 수, 특성 수)
```

예를 들어 특성이 12개라면 다음처럼 읽는다.

```text
각 샘플은 12개의 경제 지표/lag 값을 가진다.
```

모델 구조는 단순하다.

```text
입력
-> Dense
-> Dropout
-> Dense
-> 다음 달 CPI
```

## LSTM 방식

LSTM 방식은 원본 경제 지표를 시간 window로 묶는다.

예를 들어 window size가 6이면 다음 뜻이다.

```text
과거 6개월의 경제 지표를 보고 다음 달 CPI를 예측한다.
```

입력 모양은 다음과 같다.

```text
(샘플 수, 6, 경제 지표 수)
```

예를 들어 CPI, 실업률, 금리 3개를 사용하면 다음처럼 된다.

```text
(samples, 6, 3)
```

## 무엇이 더 좋을까

정답은 미리 알 수 없다.

MLP가 더 좋을 수도 있고, LSTM이 더 좋을 수도 있다.

특히 경제 데이터는 데이터 수가 많지 않은 경우가 많다. 월별 데이터는 20년을 모아도 약 240개 정도다.

이런 경우 복잡한 LSTM이 항상 유리하지 않을 수 있다.

그래서 반드시 같은 기준으로 비교해야 한다.

```text
MAE
RMSE
실제값/예측값 그래프
```

## 이번 장의 실습 흐름

```text
경제 예측 데이터셋 읽기
-> MLP용 X/y 준비
-> MLP 학습과 평가
-> LSTM용 window 데이터셋 준비
-> LSTM 학습과 평가
-> MAE/RMSE 비교
-> 결과 해석
```

## 이번 장의 과제

1. MLP가 시계열 문제를 풀 수 있는 이유를 lag feature 관점에서 설명해보자.
2. LSTM 입력 `(samples, 6, 3)`을 자기 말로 읽어보자.
3. 경제 데이터에서 LSTM이 항상 유리하지 않을 수 있는 이유를 설명해보자.
4. MAE와 RMSE 중 어떤 지표를 더 중요하게 볼지 이유와 함께 적어보자.
