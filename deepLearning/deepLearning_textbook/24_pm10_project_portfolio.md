# 24. PM10 시계열 예측 프로젝트 포트폴리오 문서 만들기

## 이번 장의 목표

19장부터 23장까지 PM10 시계열 예측 프로젝트를 진행했다.

이번 장의 목표는 다음 한 문장이다.

> PM10 예측 프로젝트를 포트폴리오에서 설명 가능한 문서로 정리한다.

시계열 프로젝트는 단순히 LSTM을 썼다고 말하면 약하다.

좋은 설명은 다음 흐름을 보여준다.

```text
시간 순서가 있는 데이터라는 점을 이해했다.
무작위 split 대신 과거로 학습하고 미래로 검증했다.
단순 baseline과 LSTM을 비교했다.
단변량 입력과 다변량 입력을 실험했다.
한계와 개선 방향을 제시했다.
```

## 프로젝트 제목

제목은 목적과 기술이 함께 보이게 적으면 좋다.

예시:

```text
서울 PM10 시간별 미세먼지 예측 LSTM 프로젝트
```

또는 조금 더 포트폴리오스럽게:

```text
환경/건강 데이터 기반 PM10 시계열 예측 프로젝트
```

## 문제 정의

문제 정의는 다음처럼 쓸 수 있다.

```text
서울 지역별 시간 단위 PM10 데이터를 사용해 다음 시간의 PM10 농도를 예측하는 시계열 회귀 모델을 구축한다.
과거 PM10 흐름을 입력으로 사용하고, LSTM 모델과 단순 baseline을 비교해 딥러닝 모델의 실효성을 확인한다.
```

여기서 중요한 단어는 “시계열”, “회귀”, “다음 시간 예측”, “baseline 비교”다.

## 데이터 설명

사용한 데이터는 다음 파일이다.

```text
C:\work\dataset\seoul_pm10.csv
```

주요 컬럼은 다음과 같다.

```text
date   = 측정 시간
area   = 서울 지역구
pm10   = 미세먼지 PM10 농도
pm2.5  = 초미세먼지 PM2.5 농도
```

초기 실험에서는 한 지역, 예를 들어 강남구만 선택해 단일 지역 시계열 예측 문제로 단순화했다.

## 전처리 설명

시계열 프로젝트의 전처리는 일반 표 데이터와 다르다.

이 프로젝트에서는 다음을 수행했다.

```text
cp949 인코딩으로 CSV 읽기
date 컬럼 datetime 변환
지역 선택
시간 순서 정렬
결측치 ffill 처리
시간 순서 기반 train/validation split
MinMaxScaler 적용
window 데이터셋 생성
```

특히 중요한 문장은 다음이다.

```text
미래 정보 누출을 막기 위해 무작위 split을 사용하지 않고, 앞쪽 80%를 학습 데이터, 뒤쪽 20%를 검증 데이터로 사용했다.
```

## 모델 설명

단변량 LSTM은 과거 PM10만 사용했다.

```text
입력 X = 과거 24시간 PM10
정답 y = 다음 시간 PM10
```

입력 모양은 다음과 같다.

```text
(samples, 24, 1)
```

다변량 LSTM은 PM2.5도 함께 사용했다.

```text
입력 X = 과거 24시간의 PM10, PM2.5
정답 y = 다음 시간 PM10
```

입력 모양은 다음과 같다.

```text
(samples, 24, 2)
```

## Baseline 비교 설명

이 프로젝트의 좋은 점은 baseline 비교다.

비교한 기준선은 다음과 같다.

```text
naive baseline = 직전 PM10 값을 다음 값으로 예측
mean baseline  = 과거 24시간 평균을 다음 값으로 예측
LSTM           = 과거 24시간 패턴을 학습해 다음 값 예측
```

이 비교를 통해 딥러닝 모델이 단순한 방법보다 실제로 나은지 확인했다.

포트폴리오 문장 예시:

```text
LSTM 모델의 성능을 해석하기 위해 naive baseline과 24시간 평균 baseline을 함께 평가했다.
이를 통해 복잡한 딥러닝 모델이 단순 기준선 대비 실제 개선을 만드는지 확인했다.
```

## 평가 지표

PM10 예측은 회귀 문제이므로 MAE와 RMSE를 사용했다.

```text
MAE  = 평균적으로 몇 PM10 정도 틀렸는가
RMSE = 큰 오차에 더 민감한 평균 오차
```

PM10 단위로 되돌려 평가하는 것이 중요하다.

```text
모델 예측값은 MinMaxScaler로 변환된 값이므로, inverse_transform으로 원래 PM10 단위로 복원한 뒤 MAE/RMSE를 계산했다.
```

## 한계

이 프로젝트의 한계는 다음과 같다.

```text
한 지역 데이터만 사용했다.
기상 정보, 계절 정보, 요일 정보 등을 사용하지 않았다.
데이터 기간이 제한적이다.
단기 예측에서는 직전 값 baseline이 매우 강할 수 있다.
LSTM이 항상 baseline보다 좋은 것은 아니다.
```

이 한계는 약점이 아니라 다음 개선 방향의 근거가 된다.

## 개선 방향

개선 방향은 다음처럼 적을 수 있다.

```text
여러 지역을 함께 사용하는 모델
기온, 습도, 풍속, 강수량 같은 기상 데이터 추가
시간대, 요일, 계절 특성 추가
GRU, 1D CNN, Transformer 계열 모델 비교
예측 시점을 1시간 뒤뿐 아니라 6시간 뒤, 24시간 뒤로 확장
Streamlit 대시보드로 시각화
```

## 이번 장의 과제

1. PM10 프로젝트의 제목을 네 말로 다시 정해보자.
2. 문제 정의를 3문장으로 작성해보자.
3. baseline 비교가 왜 중요한지 설명해보자.
4. 이 프로젝트의 한계와 개선 방향을 각각 3개 이상 적어보자.
