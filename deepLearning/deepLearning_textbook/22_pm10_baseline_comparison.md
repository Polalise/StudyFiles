# 22. PM10 예측: LSTM과 단순 Baseline 비교하기

## 이번 장의 목표

21장에서는 LSTM으로 다음 시간 PM10 값을 예측했다.

이번 장에서는 아주 중요한 질문을 던진다.

> LSTM이 단순한 기준 모델보다 정말 나은가?

딥러닝 프로젝트에서 이 질문은 매우 중요하다.

복잡한 모델을 만들었다고 해서 자동으로 좋은 프로젝트가 되는 것은 아니다. 단순한 방법보다 실제로 나은지 비교해야 한다.

## Baseline이란 무엇인가

Baseline은 비교 기준이 되는 단순한 모델이다.

딥러닝 모델의 점수를 볼 때, 기준선이 없으면 그 점수가 좋은지 나쁜지 알기 어렵다.

예를 들어 LSTM의 MAE가 8이라고 하자.

이 숫자만 보면 애매하다.

하지만 단순 baseline의 MAE가 12라면 LSTM이 어느 정도 개선했다고 볼 수 있다.

반대로 baseline의 MAE가 6이라면 LSTM은 오히려 단순 방법보다 못한 것이다.

## 직전 값 baseline

시계열 예측에서 가장 단순하면서 강력한 baseline이 있다.

```text
다음 값은 직전 값과 같을 것이다.
```

예를 들어 최근 PM10이 45였다면, 다음 시간 PM10도 45라고 예측한다.

이 방법을 naive baseline이라고 부를 수 있다.

미세먼지처럼 짧은 시간 동안 값이 급격히 변하지 않는 데이터에서는 이 baseline이 의외로 강할 수 있다.

## 평균 baseline

또 다른 단순 방법은 최근 window의 평균을 예측값으로 쓰는 것이다.

예를 들어 과거 24시간 PM10 평균이 38이라면 다음 시간도 38이라고 예측한다.

```text
직전 값 baseline = 가장 최근 값 사용
평균 baseline = 과거 window 평균 사용
```

두 방법 모두 딥러닝은 아니지만, 비교 기준으로 매우 유용하다.

## 왜 baseline 비교가 중요한가

baseline 비교는 프로젝트의 정직함을 만든다.

포트폴리오에서는 이렇게 말할 수 있어야 한다.

```text
LSTM 모델을 단순 기준선과 비교했다.
직전 값을 그대로 예측하는 naive baseline과 24시간 평균 baseline을 함께 평가했다.
이를 통해 딥러닝 모델이 실제로 예측 성능을 개선했는지 확인했다.
```

이 문장은 “LSTM을 썼습니다”보다 훨씬 좋다.

모델 선택을 근거 있게 했다는 뜻이기 때문이다.

## 비교 기준

이번 장에서는 세 가지 모델을 비교한다.

```text
1. naive baseline: 직전 PM10 값을 다음 값으로 예측
2. mean baseline: 과거 24시간 평균을 다음 값으로 예측
3. LSTM: 과거 24시간 패턴을 학습해 다음 값을 예측
```

평가 지표는 21장과 같다.

```text
MAE  = 평균적으로 몇 PM10 정도 틀렸는가
RMSE = 큰 오차에 더 민감한 평균 오차
```

## 결과 해석 방법

결과표를 볼 때는 가장 낮은 MAE와 RMSE를 찾는다.

하지만 숫자만 보지 말고 다음도 함께 생각해야 한다.

```text
LSTM이 baseline보다 얼마나 좋아졌는가?
개선 폭이 충분히 의미 있는가?
학습 시간과 복잡도를 감수할 가치가 있는가?
급격한 PM10 변화 구간에서 누가 더 잘 맞히는가?
```

만약 LSTM이 baseline보다 별로 낫지 않다면, 그것도 좋은 학습이다.

그때는 이렇게 말할 수 있다.

```text
단기 PM10 예측에서는 직전 값 baseline이 강한 기준선으로 작동했다.
LSTM의 개선 폭이 제한적이었으므로, 추가 특성이나 더 긴 기간의 데이터가 필요하다.
```

## 이번 장의 실습 흐름

```text
PM10 데이터 준비
-> window 데이터셋 생성
-> naive baseline 예측
-> mean baseline 예측
-> LSTM 예측
-> MAE/RMSE 결과표 비교
-> 예측 그래프 비교
```

## 이번 장의 과제

1. naive baseline이 무엇인지 자기 말로 설명해보자.
2. 시계열 예측에서 직전 값 baseline이 강할 수 있는 이유를 설명해보자.
3. LSTM이 baseline보다 조금만 좋다면, 포트폴리오에서 어떻게 해석해야 할지 적어보자.
4. PM10 예측 성능을 높이기 위해 추가하면 좋을 특성을 세 가지 생각해보자.
