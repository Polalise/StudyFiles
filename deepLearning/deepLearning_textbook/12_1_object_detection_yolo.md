# 12-1. 객체 탐지와 YOLO 입문

## 이번 장의 목표

앞 장들에서는 이미지를 보고 “이 사진이 어떤 종류인가?”를 맞히는 이미지 분류를 배웠다.

하지만 실제 이미지 프로젝트에서는 이런 질문도 자주 나온다.

```text
사진 안에 무엇이 있는가?
그 물체가 어디에 있는가?
물체가 몇 개 있는가?
```

이런 문제를 객체 탐지(object detection)라고 한다.

이번 장의 목표는 다음 한 문장이다.

> 이미지 분류와 객체 탐지의 차이를 이해하고, YOLO 모델이 어떤 문제를 풀기 위한 모델인지 감을 잡는다.

## 이미지 분류와 객체 탐지의 차이

이미지 분류는 이미지 전체에 하나의 이름을 붙이는 문제다.

예를 들어 꽃 사진 한 장을 보고 다음 중 하나를 고른다.

```text
daisy
dandelion
roses
sunflowers
tulips
```

출력은 보통 클래스 이름 하나다.

반면 객체 탐지는 이미지 안에서 물체의 위치까지 찾는다.

출력은 보통 다음 정보를 포함한다.

```text
물체 종류
물체 위치
모델의 확신도
```

예를 들어 이미지 안에 사람이 두 명 있고 자동차가 한 대 있다면, 객체 탐지 모델은 다음처럼 말해야 한다.

```text
person 1: 위치 상자, 확신도
person 2: 위치 상자, 확신도
car 1: 위치 상자, 확신도
```

## Bounding Box란 무엇인가

객체 탐지에서 물체 위치는 보통 사각형 상자로 표현한다.

이 사각형을 bounding box라고 부른다.

한국어로는 경계 상자라고도 한다.

```text
이미지 안의 물체를 감싸는 사각형
```

bounding box는 보통 다음 네 숫자로 표현한다.

```text
x 좌표
y 좌표
너비
높이
```

또는 다음처럼 표현하기도 한다.

```text
왼쪽 위 x
왼쪽 위 y
오른쪽 아래 x
오른쪽 아래 y
```

처음에는 “물체를 감싸는 네모 상자”로 이해하면 충분하다.

## YOLO란 무엇인가

YOLO는 객체 탐지 모델 계열의 대표적인 이름이다.

YOLO는 다음 문장의 줄임말이다.

```text
You Only Look Once
```

뜻은 “이미지를 한 번 보고 물체의 종류와 위치를 동시에 예측한다”에 가깝다.

전통적인 방식 중에는 이미지 안에서 후보 위치를 여러 번 찾고, 그 위치마다 분류를 수행하는 방식이 있었다.

YOLO는 이미지를 한 번의 신경망 통과로 처리해 빠른 객체 탐지를 목표로 한다.

## YOLO가 출력하는 것

YOLO의 출력은 이미지 분류보다 복잡하다.

이미지 분류 출력:

```text
이 이미지는 tulips일 가능성이 높다.
```

YOLO 출력:

```text
이 위치에 person이 있다. 확신도 0.91
이 위치에 car가 있다. 확신도 0.84
이 위치에 dog가 있다. 확신도 0.77
```

즉, YOLO는 다음을 동시에 예측한다.

```text
클래스
위치 상자
확신도
```

## 객체 탐지 데이터셋은 무엇이 다른가

꽃 분류 데이터셋에서는 폴더 이름이 정답이었다.

```text
roses 폴더 안 이미지 -> 정답 roses
```

하지만 객체 탐지 데이터셋에서는 이미지마다 라벨 파일이 필요하다.

라벨 파일에는 물체의 종류와 위치가 들어 있다.

예를 들어 한 이미지에 꽃이 두 송이 있다면, 라벨에는 두 개의 bounding box가 들어갈 수 있다.

```text
class_id, x_center, y_center, width, height
class_id, x_center, y_center, width, height
```

그래서 객체 탐지는 이미지 분류보다 데이터 준비가 더 어렵다.

## YOLO를 언제 고려할까

다음과 같은 문제라면 YOLO 같은 객체 탐지 모델을 고려할 수 있다.

```text
사진 안의 사람 수 세기
도로 위 차량 탐지
불량품 위치 찾기
의료 이미지에서 병변 후보 위치 찾기
매장 진열대의 상품 위치 탐지
영상에서 특정 객체 추적
```

하지만 단순히 이미지 전체의 종류만 맞히면 된다면 YOLO까지 필요하지 않다.

예를 들어 꽃 사진 한 장이 장미인지 튤립인지 맞히는 문제는 이미지 분류로 충분하다.

## YOLO와 CNN의 관계

YOLO도 내부적으로는 CNN 계열의 이미지 특징 추출을 사용한다.

다만 CNN 분류 모델은 이미지 전체의 클래스를 고르는 데 집중한다.

YOLO는 이미지 특징을 이용해 물체의 위치와 종류를 함께 예측한다.

즉, 이렇게 생각할 수 있다.

```text
CNN 분류 모델: 이미지 전체에 이름표 붙이기
YOLO 객체 탐지 모델: 이미지 안의 여러 물체에 이름표와 위치 상자 붙이기
```

## 평가 지표는 무엇이 다른가

이미지 분류에서는 accuracy를 자주 본다.

객체 탐지에서는 단순히 맞혔는지만 보면 부족하다.

물체 종류도 맞아야 하고, 위치 상자도 실제 물체 위치와 잘 겹쳐야 한다.

객체 탐지에서 자주 나오는 개념은 다음과 같다.

```text
IoU
mAP
confidence
```

처음에는 이렇게 이해하면 된다.

```text
IoU = 예측 상자와 실제 상자가 얼마나 겹치는가
mAP = 여러 클래스와 여러 기준에서 탐지 성능을 종합한 점수
confidence = 모델이 그 탐지를 얼마나 확신하는가
```

이 지표들은 객체 탐지 프로젝트에서 따로 더 깊게 배워야 한다.

## 포트폴리오에서 YOLO를 쓸 때 주의할 점

YOLO 프로젝트는 시각적으로 강력하다.

하지만 준비할 것이 많다.

```text
탐지할 객체 정의
이미지 수집
bounding box 라벨링
학습/검증 데이터 분리
모델 학습 또는 사전학습 모델 활용
탐지 결과 시각화
mAP 또는 예측 예시 정리
```

초보 포트폴리오에서는 직접 YOLO를 처음부터 학습하기보다, 사전학습된 YOLO 모델을 사용해 탐지 결과를 확인하고, 그 다음 작은 커스텀 데이터로 확장하는 흐름이 현실적이다.

## 이번 장의 과제

1. 이미지 분류와 객체 탐지의 차이를 자기 말로 설명해보자.
2. bounding box가 무엇인지 설명해보자.
3. YOLO가 출력하는 세 가지 정보를 적어보자.
4. 네가 YOLO로 만들어보고 싶은 프로젝트 주제를 하나 적어보자.

## 다음 학습 방향

YOLO를 실제로 다루려면 다음을 추가로 배워야 한다.

```text
객체 탐지 라벨 형식
bounding box 좌표
IoU
mAP
사전학습 YOLO 모델 사용법
커스텀 데이터셋 라벨링
탐지 결과 시각화
```

이 교재의 본 흐름에서는 CNN과 전이학습까지만 실습하고, YOLO는 객체 탐지 개념 보강장으로 정리한다.
