# 16. 당뇨병 예측 모델 해석하기

## 이번 장의 목표

15장에서는 여러 모델을 실험하고 결과를 비교했다.

이번 장에서는 모델의 점수를 조금 더 이해하려고 한다.

목표는 다음 한 문장이다.

> 모델이 어떤 입력 특성에 민감하게 반응하는지 확인하고, 예측 결과를 사람에게 설명할 수 있는 형태로 바꾼다.

딥러닝 모델은 종종 블랙박스처럼 느껴진다. 입력을 넣으면 결과는 나오지만, 왜 그런 결과가 나왔는지 알기 어렵다.

하지만 프로젝트에서는 최소한 다음 질문에 답할 수 있어야 한다.

```text
어떤 특성이 예측에 많이 영향을 주는가?
특정 사람의 입력값을 바꾸면 예측 점수가 어떻게 달라지는가?
모델 해석을 어디까지 믿을 수 있는가?
```

## 모델 해석은 왜 중요한가

건강 데이터 프로젝트에서는 단순히 정확도가 높다고 충분하지 않다.

예를 들어 모델이 당뇨병 위험을 높게 예측했다고 하자.

그러면 다음 질문이 자연스럽게 나온다.

```text
왜 높게 봤는가?
BMI 때문인가?
고혈압 여부 때문인가?
나이 구간 때문인가?
전반적 건강 상태 때문인가?
```

이 질문에 완벽하게 답하기는 어렵다. 하지만 모델이 어떤 특성에 민감한지 관찰할 수는 있다.

## 해석과 인과의 차이

이 장에서 아주 중요한 주의점이 있다.

모델 해석은 인과관계 증명이 아니다.

예를 들어 어떤 특성이 중요하게 나왔다고 해서 “그 특성이 반드시 질병의 원인이다”라고 말하면 안 된다.

올바른 표현은 다음에 가깝다.

```text
이 데이터와 이 모델에서는 해당 특성이 예측에 큰 영향을 주는 것으로 관찰되었다.
```

잘못된 표현은 다음과 같다.

```text
이 특성이 당뇨병의 원인이다.
```

포트폴리오에서는 이 차이를 지켜야 한다.

## Permutation Importance란 무엇인가

Permutation Importance는 특성 중요도를 보는 간단한 방법이다.

아이디어는 이렇다.

```text
1. 모델의 원래 성능을 측정한다.
2. 특정 컬럼 하나의 값을 마구 섞는다.
3. 다시 성능을 측정한다.
4. 성능이 많이 떨어지면 그 컬럼은 중요했을 가능성이 크다.
```

비유하면 시험지를 풀 때 계산기를 빼앗아 보는 것과 비슷하다.

계산기를 빼앗았을 때 점수가 크게 떨어지면, 그 사람은 계산기에 많이 의존하고 있었다고 볼 수 있다.

모델도 마찬가지다.

어떤 컬럼을 섞었을 때 점수가 크게 떨어지면, 모델은 그 컬럼에 많이 의존하고 있었을 가능성이 있다.

## 컬럼을 섞는다는 뜻

예를 들어 BMI 컬럼이 다음과 같다고 해보자.

```text
사람 A: BMI 26
사람 B: BMI 31
사람 C: BMI 22
```

BMI 컬럼만 섞으면 이렇게 될 수 있다.

```text
사람 A: BMI 31
사람 B: BMI 22
사람 C: BMI 26
```

다른 컬럼은 그대로 두고 BMI만 엉망으로 만든다.

그랬더니 모델 성능이 많이 떨어진다면, BMI는 모델에게 중요한 정보였다고 볼 수 있다.

## 한 사람의 입력값 바꿔보기

Permutation Importance는 전체 데이터 기준의 해석이다.

반면 한 사람의 입력값을 바꿔보는 방식은 개별 예측을 이해하는 데 도움이 된다.

예를 들어 한 샘플에서 BMI만 조금 바꿔보며 예측 점수를 관찰할 수 있다.

```text
BMI 22 -> 예측 점수 0.31
BMI 27 -> 예측 점수 0.46
BMI 32 -> 예측 점수 0.61
BMI 37 -> 예측 점수 0.72
```

이렇게 보면 모델이 BMI 변화에 어떻게 반응하는지 감을 잡을 수 있다.

단, 이것도 인과관계 증명이 아니다. “모델 반응 관찰”이다.

## 이번 장의 실습 흐름

```text
데이터 준비
-> MLP 모델 학습
-> 기준 F1 점수 계산
-> 컬럼 하나씩 섞어 성능 하락 확인
-> 중요도 표와 그래프 만들기
-> 특정 샘플의 입력값을 바꿔 예측 점수 변화 보기
-> 해석 문장 작성
```

## 포트폴리오에서의 설명

포트폴리오에서는 이렇게 설명할 수 있다.

```text
모델 성능뿐 아니라 예측 근거를 이해하기 위해 permutation importance를 적용했다.
각 특성을 하나씩 섞어 검증 성능이 얼마나 하락하는지 비교했고,
모델이 민감하게 사용하는 건강 지표를 확인했다.
다만 중요도는 인과관계가 아니라 모델의 예측 의존도 관찰로 해석했다.
```

이 마지막 문장이 중요하다.

“인과가 아니라 예측 의존도”라고 말할 수 있으면 훨씬 성숙한 프로젝트가 된다.

## 이번 장의 과제

1. Permutation Importance를 자기 말로 설명해보자.
2. 중요도가 높다는 말이 왜 원인이라는 뜻은 아닌지 설명해보자.
3. 모델 해석 결과를 포트폴리오에 쓸 때 조심해야 할 표현을 세 가지 적어보자.
4. 특정 사람의 BMI를 바꾸며 예측 점수를 보는 실험이 어떤 의미인지 설명해보자.
