# 14-2. Autoencoder 이상 탐지

## 왜 14장 뒤에 이 장을 배우는가

14장에서는 threshold를 배웠다.

13장의 당뇨병 예측 모델은 0과 1 사이의 확률을 출력했다.
우리는 그 확률을 0 또는 1로 바꾸기 위해 기준선을 정했다.

```text
예측 확률 >= threshold -> 1
예측 확률 < threshold  -> 0
```

이번 장에서도 threshold를 사용한다.
다만 기준이 되는 숫자가 다르다.

14장에서는 예측 확률에 threshold를 걸었다.
이번 장에서는 복원 오차에 threshold를 건다.

```text
14장:
예측 확률이 threshold보다 크면 위험으로 판단

14-a장:
복원 오차가 threshold보다 크면 이상으로 판단
```

이 차이를 이해하면 이상 탐지 문제를 훨씬 쉽게 받아들일 수 있다.

## 이상 탐지는 어떤 문제인가

이상 탐지는 평소와 다른 데이터를 찾는 문제다.

예를 들면 다음과 같다.

```text
기계 센서에서 평소와 다른 진동 패턴 찾기
카드 결제에서 평소와 다른 거래 찾기
건강 지표에서 평소와 다른 위험 신호 찾기
네트워크 로그에서 공격 의심 패턴 찾기
```

이상 탐지의 어려운 점은 정답 라벨이 부족한 경우가 많다는 것이다.

지도학습 분류는 보통 이런 데이터가 필요하다.

```text
정상 데이터 + 이상 데이터 + 각각의 정답 라벨
```

하지만 현실에서는 이상 데이터가 적거나, 어떤 것이 이상인지 아직 모를 수 있다.

그래서 이런 접근이 필요해진다.

```text
정상 데이터의 패턴을 먼저 배운다.
정상 패턴과 많이 다르면 이상으로 의심한다.
```

Autoencoder는 이 흐름에 잘 맞는 모델이다.

## Autoencoder란 무엇인가

Autoencoder는 입력을 다시 복원하도록 학습하는 신경망이다.

보통 모델은 입력을 보고 정답을 맞힌다.

```text
입력 X -> 정답 y
```

Autoencoder는 조금 다르다.

```text
입력 X -> 다시 X를 출력
```

처음 보면 이상하다.

```text
이미 입력을 알고 있는데 왜 다시 출력하게 할까?
```

이유는 모델이 데이터를 압축했다가 복원하는 과정에서 정상 패턴을 배우기 때문이다.

## 압축과 복원

Autoencoder는 크게 두 부분으로 생각할 수 있다.

```text
Encoder: 입력을 작은 표현으로 압축한다.
Decoder: 압축된 표현을 다시 원래 입력처럼 복원한다.
```

비유하면 요약과 복원이다.

긴 글을 짧게 요약한 뒤, 그 요약만 보고 원래 내용을 다시 써보는 것과 비슷하다.

정상 데이터만 많이 보고 연습하면, 모델은 정상 데이터의 규칙을 잘 복원하게 된다.

하지만 평소와 많이 다른 이상 데이터가 들어오면 잘 복원하지 못할 수 있다.
그때 원본과 복원본의 차이가 커진다.

이 차이를 복원 오차라고 한다.

```text
복원 오차 = 원래 입력과 복원된 입력의 차이
```

## 복원 오차와 threshold

이번 장의 핵심은 이 문장이다.

```text
복원 오차가 크면 이상일 가능성이 높다.
```

하지만 “크다”는 말은 기준이 필요하다.
그래서 threshold를 정한다.

예를 들어 정상 데이터의 복원 오차를 봤더니 대부분 0.1 아래에 있고, 일부만 0.3 근처까지 간다고 하자.
그러면 threshold를 정상 데이터 오차의 95% 지점으로 둘 수 있다.

```text
정상 데이터 복원 오차의 95% 지점 = threshold
복원 오차 > threshold -> 이상으로 판단
```

여기서 14장에서 배운 precision과 recall이 다시 중요해진다.

threshold를 낮추면 더 많이 이상이라고 말한다.

```text
장점: 실제 이상을 놓칠 가능성이 줄어든다.
단점: 정상도 이상이라고 오해할 수 있다.
```

threshold를 높이면 더 조심스럽게 이상이라고 말한다.

```text
장점: 이상이라고 말한 것의 신뢰가 올라갈 수 있다.
단점: 실제 이상을 놓칠 수 있다.
```

즉 14장의 trade-off가 그대로 반복된다.

## 이번 장의 실습 데이터

이번 노트북에서는 파일이 없어도 실행될 수 있도록 작은 센서 데이터를 직접 만든다.

세 가지 센서 값이 있다고 하자.

```text
temperature
pressure
vibration
```

정상 데이터는 일정한 범위 주변에 모이게 만들고,
이상 데이터는 온도, 압력, 진동이 평소보다 많이 벗어나게 만든다.

실제 프로젝트에서는 이런 synthetic 데이터보다 실제 센서 데이터나 건강 지표 데이터를 사용해야 한다.
하지만 처음 원리를 이해하기에는 작은 예제가 좋다.

## 이번 장에서 반드시 기억할 문장

```text
Autoencoder는 입력을 다시 복원하도록 학습하고, 복원 오차가 큰 데이터를 이상으로 의심한다.
```

하나 더 기억하자.

```text
이상 탐지에서도 threshold를 어떻게 정하느냐에 따라 precision과 recall의 균형이 달라진다.
```

## 확인 질문

1. Autoencoder는 입력 X를 보고 무엇을 출력하도록 학습하는가?
2. Encoder와 Decoder의 역할은 각각 무엇인가?
3. 복원 오차는 무엇인가?
4. 복원 오차가 큰 데이터를 이상으로 의심하는 이유는 무엇인가?
5. threshold를 낮추면 precision과 recall은 어떤 방향으로 변할 수 있는가?
