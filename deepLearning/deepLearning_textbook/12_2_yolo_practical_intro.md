# 12-2. YOLO 실제 실행 입문

## 왜 12-1장 뒤에 이 장을 배우는가

12-1장에서는 이미지 분류와 객체 탐지의 차이를 배웠다.

이미지 분류는 이미지 전체에 이름표 하나를 붙인다.

```text
이 이미지는 tulips이다.
```

객체 탐지는 이미지 안의 물체를 찾고, 그 위치까지 알려준다.

```text
person이 이 위치에 있다. confidence 0.91
car가 이 위치에 있다. confidence 0.84
```

이번 장에서는 이 개념이 실제 코드에서 어떤 형태로 나오는지 본다.

중요한 목표는 모델을 새로 학습하는 것이 아니다.
처음부터 YOLO를 직접 학습하려면 데이터 라벨링, bounding box, mAP, GPU 환경 같은 준비가 많이 필요하다.

이번 장의 목표는 더 작고 현실적이다.

```text
사전학습 YOLO 모델을 불러온다.
이미지 한 장에 대해 탐지를 실행한다.
결과에서 class, confidence, bounding box를 읽는다.
```

## 사전학습 모델이란 무엇인가

사전학습 모델은 이미 많은 이미지로 학습된 모델이다.

예를 들어 YOLO 사전학습 모델은 사람, 자동차, 강아지, 의자 같은 일반 물체를 어느 정도 찾을 수 있다.

처음 배우는 단계에서는 직접 학습보다 사전학습 모델 사용이 훨씬 현실적이다.

```text
직접 학습:
이미지 수집 -> 박스 라벨링 -> 학습 -> 평가 -> 수정

사전학습 모델 사용:
모델 불러오기 -> 이미지 넣기 -> 결과 확인
```

이번 장은 두 번째 흐름이다.

## ultralytics 라이브러리

Python에서 YOLO를 쉽게 써볼 수 있는 대표 라이브러리 중 하나가 `ultralytics`다.

설치되어 있지 않다면 터미널에서 다음 명령을 실행할 수 있다.

```bash
pip install ultralytics opencv-python
```

다만 이 교재의 노트북은 설치되어 있지 않아도 전체 실행이 실패하지 않도록 구성한다.
설치되어 있으면 실제 YOLO 탐지를 실행하고, 설치되어 있지 않으면 결과 구조를 설명하는 대체 예시를 보여준다.

## YOLO 모델 불러오기

YOLO 모델은 보통 다음처럼 불러온다.

```python
from ultralytics import YOLO

model = YOLO("yolov8n.pt")
```

여기서 `yolov8n.pt`는 사전학습된 YOLO 모델 파일이다.

`n`은 nano를 뜻한다.
YOLO 계열 중 작은 모델이라 CPU 환경에서도 비교적 가볍게 시험해볼 수 있다.
물론 CPU에서는 여전히 느릴 수 있다.

처음 실행할 때 모델 파일을 다운로드할 수 있으므로 인터넷 연결이 필요할 수 있다.

## predict 결과에서 읽어야 할 것

YOLO의 `predict()` 결과에서 초보자가 먼저 볼 것은 세 가지다.

```text
class: 어떤 물체인가
confidence: 모델이 얼마나 확신하는가
box: 물체가 어디에 있는가
```

예를 들어 결과가 다음과 같다면:

```text
class=person, confidence=0.87, box=[120, 50, 240, 360]
```

이 말은 다음 뜻이다.

```text
모델은 이미지 안에서 사람을 찾았다.
확신도는 0.87이다.
사람이 있는 위치는 x1=120, y1=50, x2=240, y2=360 박스다.
```

처음에는 좌표를 완벽히 외우지 않아도 된다.

```text
x1, y1 = 왼쪽 위 점
x2, y2 = 오른쪽 아래 점
```

이 정도로 이해하면 충분하다.

## confidence를 해석할 때 주의할 점

confidence가 0.90이라고 해서 무조건 정답이라는 뜻은 아니다.
모델이 그렇게 확신한다는 뜻이다.

현실 프로젝트에서는 confidence 기준을 정해야 한다.

```text
0.25 이상이면 일단 보여주기
0.50 이상만 결과로 인정하기
0.80 이상만 강한 탐지로 보기
```

기준은 프로젝트에 따라 달라진다.
의료나 안전처럼 위험이 큰 문제에서는 더 조심해야 한다.

## 이번 장의 실습 범위

이번 장의 노트북은 다음 흐름으로 구성한다.

```text
1. 샘플 이미지 경로를 준비한다.
2. ultralytics 설치 여부를 확인한다.
3. 설치되어 있으면 YOLO 모델을 불러와 탐지를 실행한다.
4. 탐지 결과의 class, confidence, box를 표로 정리한다.
5. 설치되어 있지 않으면 같은 형태의 예시 결과를 읽는 연습을 한다.
```

즉 어떤 환경에서도 최소한 결과 구조를 배울 수 있게 만든다.

## 이번 장에서 반드시 기억할 문장

```text
YOLO의 결과는 이미지 하나의 이름이 아니라, 물체마다 class, confidence, bounding box를 함께 반환한다.
```

하나 더 기억하자.

```text
초보 포트폴리오에서는 YOLO를 처음부터 학습하기보다, 사전학습 모델로 결과를 확인하고 작은 커스텀 데이터로 확장하는 흐름이 현실적이다.
```
