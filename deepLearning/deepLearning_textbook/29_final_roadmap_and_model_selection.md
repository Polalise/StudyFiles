# 29. 최종 로드맵: 상황에 맞게 딥러닝을 고르는 법

## 이번 장의 목표

여기까지 오면서 여러 종류의 데이터를 다뤘다.

```text
숫자 규칙 데이터
표 데이터
이미지 데이터
건강 위험 예측
PM10 시계열
경제 시계열
```

이번 장의 목표는 다음 한 문장이다.

> 새로운 프로젝트를 만났을 때 어떤 문제인지 판단하고, 어떤 모델과 평가 방법을 고려해야 하는지 스스로 고를 수 있게 된다.

딥러닝을 잘한다는 것은 모델 이름을 많이 외우는 것이 아니다.

중요한 것은 문제를 보고 다음 질문을 던질 수 있는 것이다.

```text
입력 데이터는 어떤 모양인가?
정답은 무엇인가?
분류인가 회귀인가?
시간 순서가 중요한가?
이미지인가 표 데이터인가?
어떤 baseline과 비교해야 하는가?
어떤 지표로 평가해야 하는가?
포트폴리오에서 어떻게 설명할 것인가?
```

## 1단계: 문제 유형부터 고른다

먼저 정답의 형태를 본다.

```text
정답이 숫자다        -> 회귀
정답이 0 또는 1이다 -> 이진 분류
정답이 여러 종류다  -> 다중 분류
정답이 이미지다      -> 생성 또는 복원
정답이 박스 위치다   -> 객체 탐지
```

예시:

```text
다음 시간 PM10 예측      -> 회귀
당뇨병 있음/없음 예측    -> 이진 분류
꽃 5종 분류              -> 다중 분류
```

## 2단계: 데이터 모양을 본다

입력 데이터의 모양은 모델 선택에 큰 영향을 준다.

```text
표 데이터      -> MLP부터 시작
이미지 데이터  -> CNN 또는 전이학습
시간 데이터    -> baseline, MLP lag, LSTM/GRU
텍스트 데이터  -> 임베딩, RNN, Transformer 계열
```

처음에는 가장 단순한 모델부터 시작하는 것이 좋다.

복잡한 모델은 baseline과 비교할 때 의미가 생긴다.

## 3단계: baseline을 만든다

baseline은 비교 기준이다.

baseline 없이 딥러닝 모델만 만들면 성능이 좋은지 알기 어렵다.

예시:

```text
이미지 분류      -> 단순 CNN과 전이학습 비교
건강 위험 예측   -> baseline MLP와 개선 MLP 비교
PM10 예측       -> 직전 값 baseline, 평균 baseline, LSTM 비교
경제 예측       -> lag MLP와 LSTM 비교
```

좋은 프로젝트는 모델 하나로 끝나지 않는다.

```text
기준선
-> 개선 모델
-> 같은 지표로 비교
-> 해석
-> 한계
-> 다음 개선 방향
```

## 4단계: 평가 지표를 고른다

문제 유형에 따라 지표가 달라진다.

```text
이진 분류:
accuracy, precision, recall, f1-score, confusion matrix

다중 분류:
accuracy, class-wise accuracy, confusion matrix

회귀:
MAE, RMSE, 실제값/예측값 그래프

시계열 회귀:
MAE, RMSE, baseline 비교, 시간 구간별 오차
```

건강 위험 예측처럼 놓치면 위험한 문제가 있다면 recall을 더 중요하게 볼 수 있다.

경제 예측처럼 큰 오차가 중요한 문제라면 RMSE를 함께 본다.

## 5단계: 데이터 누출을 막는다

데이터 누출은 모델이 실제로는 알 수 없는 정보를 학습 때 몰래 보는 문제다.

특히 다음 상황을 조심해야 한다.

```text
검증 데이터까지 포함해 scaler를 fit하는 경우
시계열 데이터를 무작위로 섞는 경우
미래 값을 입력 특성에 넣는 경우
정답과 사실상 같은 컬럼을 입력에 넣는 경우
```

안전한 기본 규칙:

```text
train 데이터로만 fit
validation/test 데이터에는 transform만
시계열은 과거로 학습하고 미래로 검증
target은 입력 X에 넣지 않기
```

## 6단계: 포트폴리오 문장으로 바꾼다

프로젝트는 결과만 있는 것이 아니라 설명이 있어야 한다.

좋은 포트폴리오 문서는 다음 구조를 가진다.

```text
1. 문제 정의
2. 데이터 설명
3. 전처리
4. 모델 선택 이유
5. baseline 비교
6. 평가 지표
7. 결과 해석
8. 한계
9. 개선 방향
10. 데모 또는 재사용 가능 구조
```

이 구조를 따르면 프로젝트가 훨씬 단단해 보인다.

## 프로젝트별 최종 정리

### 꽃 이미지 분류

```text
데이터 모양: 이미지
문제 유형: 다중 분류
모델 후보: CNN, MobileNetV2 전이학습
평가: accuracy, confusion matrix, 오답 이미지 확인
포트폴리오 포인트: 전이학습과 오답 분석
```

### 당뇨병 위험 예측

```text
데이터 모양: 표 데이터
문제 유형: 이진 분류
모델 후보: MLP
평가: accuracy, precision, recall, f1-score
포트폴리오 포인트: threshold 조정, 모델 해석, 진단이 아닌 위험 예측 표현
```

### PM10 예측

```text
데이터 모양: 시간별 시계열
문제 유형: 회귀
모델 후보: naive baseline, mean baseline, LSTM, 다변량 LSTM
평가: MAE, RMSE, 실제값/예측값 그래프
포트폴리오 포인트: baseline 비교, 시간 순서 split, PM2.5 추가 실험
```

### 경제 시계열 예측

```text
데이터 모양: 월별 경제 시계열
문제 유형: 회귀
모델 후보: lag MLP, LSTM
평가: MAE, RMSE
포트폴리오 포인트: API 수집, target/lag 설계, 미래 정보 누출 방지
```

## 최종 프로젝트 선택 가이드

취업용 포트폴리오로 가장 추천하는 흐름은 다음 중 하나다.

```text
1순위: 당뇨병 위험 예측 프로젝트
장점: 표 데이터, 평가 지표, threshold, 해석, 데모 연결이 모두 가능하다.

2순위: PM10 시계열 예측 프로젝트
장점: 환경/건강 주제이며 baseline 비교와 시계열 split을 보여주기 좋다.

3순위: 꽃 이미지 전이학습 프로젝트
장점: 시각적으로 보여주기 좋고 데모가 쉽다.

4순위: 경제 시계열 예측 프로젝트
장점: API 수집과 시계열 설계가 강점이지만, 해석과 예측 표현에 주의가 필요하다.
```

가장 좋은 선택은 하나만 고르는 것이 아니라, 대표 프로젝트 하나를 깊게 만들고 나머지는 보조 프로젝트로 정리하는 것이다.

## 앞으로의 학습 순서

이 교재 이후에는 다음 순서로 가면 좋다.

```text
1. 당뇨병 프로젝트 하나를 완전히 실행한다.
2. 결과표와 그래프를 저장한다.
3. README를 자기 말로 수정한다.
4. Streamlit 데모를 만든다.
5. GitHub 포트폴리오 구조로 정리한다.
6. 발표용 1분 요약을 연습한다.
7. PM10 또는 꽃 프로젝트를 두 번째 프로젝트로 정리한다.
```

## 마지막 과제

1. 네가 최종 포트폴리오 대표 프로젝트로 삼고 싶은 주제를 하나 고르자.
2. 그 프로젝트의 문제 정의를 3문장으로 쓰자.
3. 사용할 평가 지표를 고르고 이유를 적자.
4. baseline을 무엇으로 둘지 정하자.
5. 프로젝트의 한계를 최소 3개 적자.
6. 면접에서 말할 1분 요약을 작성하자.

## 마무리

딥러닝은 모델을 외우는 공부가 아니다.

데이터를 보고, 문제를 정의하고, 기준선을 만들고, 모델을 비교하고, 결과를 해석하고, 한계를 말하는 과정이다.

이 과정을 할 수 있으면 새로운 문제를 만나도 겁이 줄어든다.

이제 목표는 “모든 장을 읽었다”가 아니라, 하나의 프로젝트를 끝까지 완성하는 것이다.
