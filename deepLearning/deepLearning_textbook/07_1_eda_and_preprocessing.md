# 07. EDA와 전처리

## 이번 장에서 배울 것

모델을 만들기 전에 데이터를 먼저 봐야 한다.

이번 장에서는 다음을 배운다.

```text
EDA: 데이터를 눈으로 확인하는 과정
전처리: 모델이 먹기 좋은 형태로 데이터를 정리하는 과정
```

딥러닝 모델은 데이터를 보고 배운다. 그래서 데이터가 이상하면 모델도 이상하게 배운다.

## EDA란 무엇인가

EDA는 Exploratory Data Analysis의 줄임말이다. 한국어로는 탐색적 데이터 분석이라고 한다.

쉽게 말하면 다음 질문에 답하는 과정이다.

```text
데이터가 몇 줄인가?
컬럼은 무엇이 있는가?
정답 컬럼은 무엇인가?
비어 있는 값은 있는가?
숫자 컬럼과 문자 컬럼은 무엇인가?
값이 한쪽으로 치우쳐 있지는 않은가?
```

## 왜 모델보다 데이터를 먼저 봐야 할까

모델은 데이터에서 규칙을 찾는다. 그런데 데이터에 문제가 있으면 모델은 잘못된 규칙을 배울 수 있다.

예를 들어 심장 질환 예측 데이터에서 `Age`가 비어 있거나, 혈압이 말도 안 되는 값으로 들어 있으면 모델은 혼란스러워진다.

그래서 모델을 만들기 전에 데이터를 먼저 확인해야 한다.

## 자주 하는 EDA

가장 기본적인 EDA는 다음과 같다.

```python
df.head()
df.shape
df.info()
df.isnull().sum()
df.describe()
df["target"].value_counts()
```

각각의 뜻은 다음과 같다.

```text
head(): 앞부분 몇 줄 보기
shape: 행과 열 개수 보기
info(): 컬럼 이름, 타입, 결측치 정보 보기
isnull().sum(): 비어 있는 값 개수 보기
describe(): 숫자 컬럼의 평균, 최솟값, 최댓값 보기
value_counts(): 값별 개수 보기
```

## 전처리란 무엇인가

전처리는 데이터를 모델이 이해할 수 있게 정리하는 과정이다.

대표적인 전처리는 다음과 같다.

```text
입력 X와 정답 y 분리
문자 데이터를 숫자로 바꾸기
결측치 처리
스케일링
학습/검증 데이터 나누기
```

## 입력 X와 정답 y

모델 학습에는 입력과 정답이 필요하다.

```text
X: 모델이 보는 입력
y: 모델이 맞혀야 하는 정답
```

심장 질환 예측이라면:

```text
X: 나이, 성별, 혈압, 콜레스테롤 등
y: 심장 질환 여부
```

## 문자 데이터를 숫자로 바꾸기

딥러닝 모델은 문자를 그대로 이해하지 못한다.

예를 들어 성별이 `M`, `F`로 되어 있으면 숫자로 바꿔야 한다.

```text
M -> 1
F -> 0
```

또는 원핫 인코딩을 사용할 수 있다.

```text
ChestPainType = ATA
-> ChestPainType_ATA = 1
-> ChestPainType_NAP = 0
-> ChestPainType_ASY = 0
```

## 스케일링

스케일링은 숫자의 범위를 비슷하게 맞추는 과정이다.

예를 들어 나이는 20~80 정도인데, 콜레스테롤은 100~600 정도일 수 있다.

모델이 특정 큰 숫자 컬럼에 과하게 영향을 받지 않도록 숫자 범위를 조정한다.

자주 쓰는 도구:

```text
StandardScaler
MinMaxScaler
```

입문 단계에서는 이렇게 기억하자.

```text
StandardScaler: 평균 0, 표준편차 1 근처로 맞춤
MinMaxScaler: 0과 1 사이로 맞춤
```

## 데이터 누수 주의

전처리에서 가장 중요한 주의점이 있다.

```text
scaler는 학습 데이터에만 fit한다.
검증/테스트 데이터에는 transform만 한다.
```

나중에 평가할 데이터를 미리 보고 스케일 기준을 잡으면 시험 문제 답을 미리 본 것과 비슷해진다. 이것을 데이터 누수라고 한다.

## 이번 장에서 반드시 기억할 문장

```text
모델을 만들기 전에 데이터를 먼저 보고, 입력 X와 정답 y를 분리하고, 모델이 이해할 수 있는 숫자 형태로 정리해야 한다.
```
