# 16-2. PCA와 SHAP 입문

## 왜 16장 뒤에 이 장을 배우는가

16장에서는 모델 해석을 처음 배웠다.
핵심 도구는 permutation importance였다.

Permutation importance의 질문은 단순했다.

```text
어떤 특성 하나를 섞었을 때 모델 성능이 많이 떨어지는가?
```

성능이 많이 떨어진다면, 그 특성은 모델이 중요하게 사용했을 가능성이 있다.

이번 장에서는 원본 예제에 있던 두 가지 도구를 추가로 만난다.

```text
PCA
SHAP
```

둘 다 데이터 분석과 모델 설명에서 자주 보이지만, 목적이 다르다.

이 차이를 먼저 분명히 해야 한다.

```text
PCA: 데이터 구조를 보기 위한 차원 축소 도구
SHAP: 모델 예측을 설명하기 위한 해석 도구
```

PCA는 모델이 왜 그런 예측을 했는지 직접 설명하는 도구가 아니다.
SHAP는 모델의 예측에 각 특성이 얼마나 기여했는지 설명하려는 도구다.

## PCA가 필요한 이유

건강 데이터에는 특성이 많다.

예를 들어 다음과 같은 열이 있을 수 있다.

```text
BMI
Age
HighBP
HighChol
PhysActivity
GenHlth
MentHlth
PhysHlth
...
```

사람은 2차원 그래프를 보기는 쉽다.
하지만 특성이 10개, 20개가 되면 눈으로 보기 어렵다.

PCA는 많은 특성을 몇 개의 축으로 압축해 보여주는 방법이다.

예를 들어 특성 20개를 2개 축으로 줄이면 산점도를 그릴 수 있다.

```text
원래 데이터: 특성 20개
PCA 결과: PC1, PC2 두 축
```

여기서 PC는 Principal Component의 줄임말이다.
한국어로는 주성분이라고 부른다.

## PCA를 해석할 때 주의할 점

PCA 그래프에서 클래스가 어느 정도 나뉘어 보일 수 있다.
하지만 이것이 곧 좋은 딥러닝 모델을 의미하지는 않는다.

PCA는 정답을 맞히기 위해 학습하는 모델이 아니다.
데이터의 큰 변동 방향을 찾아 2차원으로 눌러 보여주는 도구다.

그래서 PCA는 이렇게 쓰는 편이 안전하다.

```text
데이터가 대략 어떤 모양인지 본다.
이상하게 동떨어진 점이 있는지 본다.
클래스가 완전히 섞여 있는지, 조금 나뉘는지 감을 잡는다.
```

반대로 이렇게 말하면 위험하다.

```text
PCA에서 분리되어 보이므로 이 특성이 원인이다.
PCA 그래프가 좋으므로 모델도 반드시 좋다.
```

16장에서 배운 것처럼 해석과 인과는 다르다.

## SHAP가 필요한 이유

Permutation importance는 전체 데이터 기준의 중요도를 보는 데 좋다.

하지만 이런 질문에는 약하다.

```text
이 한 사람의 예측 점수가 높게 나온 이유는 무엇인가?
```

SHAP는 이런 개별 예측 설명에 자주 쓰인다.

SHAP의 핵심 질문은 다음과 같다.

```text
각 특성이 이 예측값을 얼마나 올리거나 내렸는가?
```

예를 들어 어떤 사람의 위험 점수가 높게 나왔다고 하자.
SHAP는 대략 이런 식의 설명을 목표로 한다.

```text
BMI는 위험 점수를 올리는 방향으로 작용했다.
Age도 위험 점수를 올리는 방향으로 작용했다.
PhysActivity는 위험 점수를 낮추는 방향으로 작용했다.
```

물론 이것도 인과관계 증명은 아니다.
모델이 그렇게 사용했다는 설명에 가깝다.

## permutation importance와 SHAP의 차이

둘을 단순히 비교하면 다음과 같다.

```text
Permutation importance:
특성을 섞었을 때 전체 성능이 얼마나 떨어지는가?

SHAP:
특정 예측에서 각 특성이 예측값을 어느 방향으로 얼마나 밀었는가?
```

처음에는 이렇게 기억하면 된다.

```text
Permutation importance = 전체적인 중요도
SHAP = 전체 중요도와 개별 예측 설명 모두에 쓰일 수 있는 도구
```

## 이번 장의 실습 범위

이번 노트북에서는 세 가지를 한다.

```text
1. 건강/당뇨 데이터 불러오기
2. PCA로 2차원 시각화하기
3. RandomForest 모델로 permutation importance 계산하기
4. SHAP가 설치되어 있으면 선택적으로 SHAP 값 계산하기
```

SHAP는 환경에 따라 설치가 되어 있지 않을 수 있다.
그래서 필수 실습은 permutation importance까지로 두고, SHAP는 선택 실습으로 구성한다.

## 이번 장에서 반드시 기억할 문장

```text
PCA는 데이터를 낮은 차원으로 압축해 구조를 보는 도구이고, SHAP는 모델 예측을 설명하기 위한 도구다.
```

하나 더 기억하자.

```text
PCA, permutation importance, SHAP 모두 인과관계를 증명하는 도구가 아니라 데이터와 모델의 반응을 이해하기 위한 도구다.
```
