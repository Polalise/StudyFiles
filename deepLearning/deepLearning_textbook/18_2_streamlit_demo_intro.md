# 18-2. Streamlit 데모 입문

## 왜 18장 뒤에 이 장을 배우는가

17장에서는 예측 함수를 만들었다.

그 예측 함수의 목적은 단순했다.

```text
새로운 입력값을 받는다.
학습 때와 같은 전처리를 적용한다.
모델로 예측한다.
예측 점수와 해석 문장을 돌려준다.
```

18장에서는 그 과정을 포트폴리오 문서로 정리했다.

하지만 취업용 포트폴리오에서는 문서만큼 중요한 것이 하나 더 있다.

```text
다른 사람이 직접 눌러볼 수 있는 데모
```

노트북은 분석 과정을 보여주기 좋다.
하지만 사용자가 값을 직접 넣고 결과를 확인하기에는 조금 불편하다.

Streamlit은 이런 작은 데모를 빠르게 만들 수 있는 도구다.

## Streamlit은 무엇인가

Streamlit은 Python 코드만으로 간단한 웹 앱을 만들 수 있게 도와주는 라이브러리다.

예를 들어 이런 코드를 쓰면:

```python
age = st.slider("Age", 18, 100, 45)
bmi = st.number_input("BMI", 10.0, 60.0, 25.0)
```

웹 화면에는 슬라이더와 숫자 입력창이 생긴다.

그리고 사용자가 버튼을 누르면 Python 코드가 실행되어 결과를 보여줄 수 있다.

```python
if st.button("예측하기"):
    st.metric("예측 점수", "0.72")
```

처음에는 Streamlit을 거창한 웹 개발로 생각하지 않아도 된다.

```text
노트북에서 만든 예측 코드를
사람이 눌러볼 수 있는 입력 화면으로 감싸는 도구
```

이 정도로 이해하면 충분하다.

## 노트북과 데모 앱의 차이

노트북은 만드는 사람에게 좋다.

```text
데이터 확인
그래프 그리기
모델 학습
실험 기록
중간 결과 설명
```

데모 앱은 보는 사람에게 좋다.

```text
입력값을 바꿔본다.
예측 결과를 바로 본다.
모델이 어떤 식으로 동작하는지 체험한다.
```

취업용 포트폴리오에서는 둘 다 가치가 있다.

```text
노트북 = 내가 어떻게 분석하고 만들었는지 보여줌
데모 앱 = 결과물을 어떻게 사용할 수 있는지 보여줌
```

## 17장과 연결되는 핵심

Streamlit 앱에서 가장 중요한 것은 모델 파일만 불러오는 것이 아니다.

17장에서 배운 세 가지를 함께 지켜야 한다.

```text
1. 모델
2. scaler
3. feature order
```

모델만 저장하고 scaler를 잊으면, 앱에 들어온 숫자가 학습 때와 다른 형태로 모델에 들어갈 수 있다.

feature order를 잊으면, BMI 값이 HighBP 자리로 들어가는 식의 심각한 문제가 생길 수 있다.

그래서 실제 데모 앱은 이런 흐름을 가져야 한다.

```text
사용자 입력
-> DataFrame 한 줄 만들기
-> 학습 때 사용한 feature order로 정렬
-> scaler.transform()
-> model.predict()
-> threshold 적용
-> 결과와 주의 문구 표시
```

이번 장의 앱은 아직 실제 저장 모델을 불러오지는 않는다.
대신 앱 구조를 먼저 익히기 위해 임시 위험 점수 계산식을 사용한다.

실제 프로젝트에서는 이 임시 계산 부분을 17장의 예측 함수로 바꾸면 된다.

## 의료/건강 데모에서 반드시 조심할 표현

당뇨병 위험 예측 같은 주제는 조심해야 한다.

데모 앱에서 이렇게 말하면 안 된다.

```text
당신은 당뇨병입니다.
당신은 안전합니다.
이 앱은 진단 결과입니다.
```

대신 이렇게 표현해야 한다.

```text
이 결과는 학습용 모델의 예측 점수입니다.
의학적 진단이 아닙니다.
실제 판단은 전문가 상담과 검사를 통해 이루어져야 합니다.
```

이런 문구는 포트폴리오의 신뢰도를 떨어뜨리지 않는다.
오히려 문제를 책임감 있게 다룬다는 신호가 된다.
