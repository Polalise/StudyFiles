# 15. 당뇨병 예측 모델 개선과 실험 관리

## 이번 장의 목표

13장에서는 baseline 모델을 만들었다. 14장에서는 threshold를 바꾸면 precision과 recall이 어떻게 달라지는지 봤다.

이번 장의 목표는 다음 한 문장이다.

> 모델을 여러 개 실험하고, 결과를 표로 비교해서 어떤 개선이 실제로 도움이 되었는지 판단한다.

딥러닝 프로젝트에서 중요한 것은 “한 번 학습했다”가 아니다.

중요한 것은 다음 흐름이다.

```text
기준 모델 만들기
-> 개선 아이디어 세우기
-> 실험하기
-> 같은 기준으로 평가하기
-> 결과표로 비교하기
-> 가장 나은 모델 저장하기
```

이 흐름을 할 수 있어야 포트폴리오에서 프로젝트다운 이야기가 된다.

## 실험이란 무엇인가

실험은 아무렇게나 모델을 바꿔보는 것이 아니다.

한 번에 너무 많은 것을 바꾸면 어떤 변화가 효과를 냈는지 알 수 없다.

예를 들어 다음처럼 바꾸면 해석이 어렵다.

```text
Dense 층 개수 변경
Dropout 변경
batch_size 변경
learning rate 변경
epoch 변경
threshold 변경
```

이 모든 것을 한 번에 바꾸면 결과가 좋아져도 왜 좋아졌는지 모른다.

그래서 초보 단계에서는 실험을 작게 나누는 것이 좋다.

```text
실험 A: baseline
실험 B: Dropout 조금 증가
실험 C: Dense 층 조금 더 넓게
```

## EarlyStopping이란 무엇인가

딥러닝 모델은 오래 학습한다고 항상 좋아지지 않는다.

학습 데이터에는 계속 잘 맞는데, 검증 데이터 성능이 더 이상 좋아지지 않거나 나빠질 수 있다.

이때 사용하는 도구가 `EarlyStopping`이다.

뜻은 간단하다.

> 검증 성능이 더 이상 좋아지지 않으면 학습을 일찍 멈춘다.

예를 들어 다음 설정을 사용할 수 있다.

```python
EarlyStopping(monitor="val_loss", patience=3, restore_best_weights=True)
```

읽는 법은 이렇다.

```text
monitor="val_loss"          = 검증 손실을 지켜본다
patience=3                  = 3번 연속 좋아지지 않으면 멈춘다
restore_best_weights=True   = 가장 좋았던 시점의 가중치로 되돌린다
```

## Dropout을 왜 조절할까

Dropout은 학습 중 일부 뉴런 출력을 임시로 끄는 방법이다.

모델이 특정 패턴에 너무 의존하는 것을 줄여 과적합을 완화할 수 있다.

하지만 Dropout을 너무 크게 넣으면 학습 자체가 어려워질 수 있다.

```text
Dropout이 너무 작음: 과적합을 충분히 못 막을 수 있음
Dropout이 적당함: 일반화에 도움
Dropout이 너무 큼: 모델이 충분히 배우기 어려움
```

그래서 실험으로 비교해야 한다.

## 모델을 넓힌다는 뜻

`Dense(32)`를 `Dense(64)`로 바꾸면 뉴런 수가 늘어난다.

이것을 모델을 조금 넓힌다고 볼 수 있다.

뉴런 수가 많아지면 더 복잡한 패턴을 배울 수 있지만, 과적합 위험도 커질 수 있다.

```text
작은 모델: 단순하고 빠르지만 표현력이 부족할 수 있음
큰 모델: 복잡한 패턴을 배울 수 있지만 과적합 위험 증가
```

## 결과표가 중요한 이유

실험을 여러 번 하면 기억으로 비교하기 어렵다.

그래서 결과를 표로 남긴다.

예를 들어 다음과 같은 표를 만들 수 있다.

```text
experiment_name | accuracy | precision | recall | f1 | val_loss
baseline        | ...
dropout_0.4     | ...
wider_model     | ...
```

이 표가 있어야 “어떤 모델을 최종 선택했는가”를 설명할 수 있다.

## 이번 장의 실습 흐름

```text
데이터 준비
-> 실험용 모델 생성 함수 만들기
-> baseline / dropout / wider 모델 학습
-> 같은 threshold에서 평가
-> 결과표 만들기
-> 가장 좋은 모델 저장하기
```

이번 장에서 중요한 것은 최고 점수를 내는 것이 아니다.

중요한 것은 비교 가능한 실험 구조를 만드는 것이다.

## 포트폴리오에서의 설명

포트폴리오에서는 이런 식으로 설명할 수 있다.

```text
Baseline MLP를 기준으로 Dropout과 모델 크기를 바꾼 실험을 진행했다.
모든 실험은 같은 train/validation split과 같은 평가 기준으로 비교했다.
EarlyStopping을 사용해 검증 손실이 개선되지 않을 때 학습을 멈추도록 했다.
최종 모델은 accuracy뿐 아니라 precision, recall, f1-score를 함께 고려해 선택했다.
```

## 이번 장의 과제

1. EarlyStopping이 왜 필요한지 설명해보자.
2. Dropout을 너무 크게 넣으면 어떤 문제가 생길 수 있는지 설명해보자.
3. 모델을 넓히면 어떤 장점과 단점이 있는지 설명해보자.
4. 최종 모델을 고를 때 accuracy만 보면 안 되는 이유를 적어보자.
