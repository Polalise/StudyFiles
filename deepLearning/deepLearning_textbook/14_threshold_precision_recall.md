# 14. 임계값, Precision, Recall 이해하기

## 이번 장의 목표

13장에서는 당뇨병 위험 예측 baseline 모델을 만들었다. 모델은 마지막에 `sigmoid`를 사용해서 0과 1 사이의 숫자를 출력했다.

이번 장의 목표는 다음 한 문장이다.

> 모델의 예측 확률을 0 또는 1로 바꾸는 기준인 임계값(threshold)을 이해하고, precision과 recall의 균형을 해석한다.

이 장은 건강/의료 위험 예측 프로젝트에서 아주 중요하다.

정확도 하나만 보고 “모델이 좋다”고 말하면 부족하다. 특히 건강 위험 예측에서는 “위험한 사람을 얼마나 놓치지 않는가”와 “위험하다고 말한 사람이 실제로 얼마나 맞는가”를 함께 봐야 한다.

## 예측 확률이란 무엇인가

13장의 모델 출력은 이런 숫자였다.

```text
0.12
0.47
0.51
0.83
```

이 숫자는 모델이 “정답이 1일 가능성이 어느 정도라고 보는지”를 나타낸다.

이번 프로젝트에서는 `1 = 당뇨병 있음`으로 두었다.

따라서 다음처럼 해석할 수 있다.

```text
0.12 = 모델은 당뇨병 있음 가능성을 낮게 봄
0.83 = 모델은 당뇨병 있음 가능성을 높게 봄
```

단, 이 값은 의학적 확률이 아니다. 데이터 패턴을 학습한 모델의 예측 점수다.

## 임계값이란 무엇인가

모델은 0과 1 사이의 숫자를 내지만, 최종 분류 결과는 0 또는 1이어야 한다.

그래서 기준선이 필요하다.

이 기준선을 임계값(threshold)이라고 부른다.

가장 흔한 기준은 0.5다.

```text
예측 확률 >= 0.5 -> 1로 예측
예측 확률 < 0.5  -> 0으로 예측
```

하지만 0.5가 언제나 정답은 아니다.

프로젝트 목적에 따라 임계값은 바뀔 수 있다.

## 임계값을 낮추면 어떻게 될까

임계값을 0.5에서 0.3으로 낮춘다고 해보자.

```text
0.5 기준: 0.42는 0으로 예측
0.3 기준: 0.42는 1로 예측
```

임계값을 낮추면 더 많은 사람을 “위험 있음”으로 예측한다.

장점:

```text
실제 위험이 있는 사람을 놓칠 가능성이 줄어든다.
recall이 올라갈 수 있다.
```

단점:

```text
실제 위험이 없는 사람도 위험하다고 예측할 수 있다.
precision이 내려갈 수 있다.
```

## 임계값을 높이면 어떻게 될까

임계값을 0.5에서 0.7로 높인다고 해보자.

```text
0.5 기준: 0.62는 1로 예측
0.7 기준: 0.62는 0으로 예측
```

임계값을 높이면 모델이 더 조심스럽게 1을 예측한다.

장점:

```text
위험이라고 예측한 사람 중 실제 위험인 비율이 올라갈 수 있다.
precision이 올라갈 수 있다.
```

단점:

```text
실제 위험이 있는 사람을 놓칠 수 있다.
recall이 내려갈 수 있다.
```

## Precision과 Recall 다시 보기

precision은 이렇게 읽는다.

```text
모델이 1이라고 말한 사람 중, 진짜 1은 얼마나 되는가?
```

recall은 이렇게 읽는다.

```text
진짜 1인 사람 중, 모델이 1이라고 찾아낸 사람은 얼마나 되는가?
```

건강 위험 예측에서는 recall이 중요할 수 있다. 실제 위험이 있는 사람을 놓치는 것이 문제가 될 수 있기 때문이다.

하지만 recall만 높이면 아무나 위험하다고 말하는 모델이 될 수 있다. 그래서 precision도 함께 봐야 한다.

## 혼동 행렬을 사람 말로 읽기

이진 분류의 confusion matrix는 네 칸으로 생각할 수 있다.

```text
TN = 실제 0, 예측 0
FP = 실제 0, 예측 1
FN = 실제 1, 예측 0
TP = 실제 1, 예측 1
```

건강 위험 예측에서 특히 조심해야 하는 칸은 FN이다.

```text
FN = 실제로 위험이 있는데 모델이 위험 없다고 예측한 경우
```

이런 경우가 많으면 실제 위험군을 놓치는 모델이 된다.

## 이번 장의 실습 흐름

이번 장에서는 13장의 baseline 흐름을 다시 만들고, 임계값만 바꿔가며 평가한다.

```text
데이터 읽기
-> baseline 모델 학습
-> 예측 확률 만들기
-> threshold 0.3, 0.5, 0.7 비교
-> precision, recall, f1-score 비교
-> precision-recall curve 보기
```

## 포트폴리오에서의 설명

포트폴리오에서는 이런 문장이 좋다.

```text
의료/건강 위험 예측에서는 accuracy만으로 모델을 평가하지 않고,
임계값 변화에 따른 precision과 recall의 trade-off를 분석했다.
특히 위험군을 놓치는 FN을 줄이는 것이 중요할 수 있으므로,
프로젝트 목적에 맞는 threshold 선택이 필요함을 확인했다.
```

이 문장은 “모델을 돌려봤다”가 아니라 “문제 목적에 맞게 평가했다”는 느낌을 준다.

## 이번 장의 과제

1. 임계값을 낮추면 precision과 recall이 각각 어떻게 변할 수 있는지 설명해보자.
2. 임계값을 높이면 어떤 장점과 단점이 있는지 설명해보자.
3. 건강 위험 예측에서 FN이 왜 중요할 수 있는지 설명해보자.
4. 너라면 이 프로젝트에서 threshold를 0.5보다 낮출지 높일지, 이유와 함께 적어보자.
