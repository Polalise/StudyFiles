# 17. 당뇨병 위험 예측 함수 만들기

## 이번 장의 목표

지금까지는 노트북 안에서 모델을 학습하고 평가했다.

이번 장에서는 한 단계 더 나아간다.

> 학습한 모델과 전처리 도구를 저장하고, 새 입력값을 받아 예측하는 함수를 만든다.

이 장은 데모 앱이나 포트폴리오 프로젝트로 넘어가기 위한 연결 장이다.

## 왜 모델만 저장하면 부족할까

딥러닝 모델은 입력 숫자를 그대로 보고 학습하지 않았다.

13장부터 우리는 `StandardScaler`를 사용했다.

```text
원본 건강 지표
-> StandardScaler로 변환
-> MLP 모델 입력
-> 예측 확률 출력
```

따라서 나중에 새 사람의 데이터를 예측할 때도 같은 scaler를 사용해야 한다.

모델만 저장하고 scaler를 저장하지 않으면 문제가 생긴다.

```text
학습 때 본 입력 모양과
예측 때 들어가는 입력 모양이 달라진다.
```

딥러닝 프로젝트에서는 모델 파일뿐 아니라 전처리 도구도 함께 보존해야 한다.

## feature order란 무엇인가

표 데이터 모델에서는 컬럼 순서가 매우 중요하다.

모델은 컬럼 이름을 이해하는 것이 아니라 숫자 배열의 위치를 본다.

예를 들어 학습 때 입력 순서가 이랬다고 하자.

```text
HighBP, HighChol, BMI
```

그런데 예측 때 실수로 이렇게 넣으면 문제가 된다.

```text
BMI, HighBP, HighChol
```

사람이 보기에는 컬럼 이름이 있으니 괜찮아 보일 수 있다. 하지만 모델에 들어가는 숫자 배열 순서가 바뀌면, 모델은 BMI 값을 HighBP처럼 해석할 수 있다.

그래서 feature order, 즉 학습 때 사용한 컬럼 순서를 저장해야 한다.

## 예측 함수가 해야 할 일

새 입력값을 받아 예측하는 함수는 다음 일을 해야 한다.

```text
1. 입력값을 DataFrame 한 행으로 만든다.
2. 학습 때 사용한 컬럼 순서로 정렬한다.
3. 저장된 scaler로 transform한다.
4. 저장된 모델로 predict한다.
5. threshold 기준으로 0/1 결과를 만든다.
6. 예측 확률과 해석 문장을 반환한다.
```

이 흐름을 함수로 묶어두면 데모 앱에서도 그대로 사용할 수 있다.

## joblib이란 무엇인가

Keras 모델은 `.keras` 파일로 저장할 수 있다.

하지만 `StandardScaler`나 컬럼 목록 같은 파이썬 객체는 Keras 모델 파일에 자동으로 들어가지 않는다.

이런 객체를 저장할 때 `joblib`을 자주 사용한다.

```python
joblib.dump(scaler, "scaler.joblib")
joblib.load("scaler.joblib")
```

읽는 법은 단순하다.

```text
dump = 저장한다
load = 불러온다
```

## 위험 예측 표현 주의

이번 프로젝트는 교육용이다.

따라서 예측 결과를 이렇게 표현하면 안 된다.

```text
당신은 당뇨병입니다.
```

대신 이렇게 표현해야 한다.

```text
모델이 학습한 데이터 패턴 기준으로 당뇨병 있음 클래스의 예측 점수가 높게 나왔습니다.
이 결과는 의학적 진단이 아니며, 교육용 모델 출력입니다.
```

이 차이는 매우 중요하다.

## 이번 장의 실습 흐름

```text
데이터 준비
-> scaler와 모델 학습
-> 모델 저장
-> scaler와 feature_names 저장
-> 다시 불러오기
-> 새 샘플 예측 함수 만들기
-> 예측 결과 문장 만들기
```

## 포트폴리오에서의 의미

포트폴리오에서는 “학습했다”보다 “재사용 가능한 예측 파이프라인을 만들었다”가 더 강하다.

이렇게 설명할 수 있다.

```text
학습된 MLP 모델뿐 아니라 StandardScaler와 feature order를 함께 저장했다.
새 입력값이 들어오면 학습 때와 같은 전처리를 적용한 뒤 예측하도록 함수화했다.
이를 통해 노트북 분석 결과를 데모 앱으로 확장할 수 있는 구조를 만들었다.
```

## 이번 장의 과제

1. 모델만 저장하고 scaler를 저장하지 않으면 왜 문제가 되는지 설명해보자.
2. feature order가 왜 중요한지 설명해보자.
3. `joblib.dump`와 `joblib.load`의 역할을 설명해보자.
4. 예측 결과를 의료 진단처럼 말하면 안 되는 이유를 적어보자.
