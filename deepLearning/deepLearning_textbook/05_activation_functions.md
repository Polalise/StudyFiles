# 05. 활성화 함수

## 이번 장에서 배울 것

이번 장에서는 활성화 함수를 배운다.

활성화 함수는 모델 안에서 나온 숫자를 목적에 맞게 바꿔주는 장치다.

대표적으로 다음 네 가지를 다룬다.

```text
sigmoid
tanh
ReLU
softmax
```

## 활성화 함수가 왜 필요할까

모델은 계산을 하면서 여러 숫자를 만든다. 그런데 어떤 문제에서는 출력값의 범위가 정해져 있어야 한다.

예를 들어 스팸 메일 확률을 예측한다고 하자.

```text
스팸일 확률: 0% ~ 100%
```

이 값은 음수가 되면 이상하고, 100%를 넘어도 이상하다.

그래서 어떤 숫자가 나오더라도 0과 1 사이로 눌러주는 함수가 필요하다. 이런 역할을 하는 대표 함수가 sigmoid다.

## sigmoid

sigmoid는 어떤 숫자를 넣어도 0과 1 사이 값으로 바꿔준다.

```text
아주 작은 값 -> 0에 가까움
0 근처 값 -> 0.5 근처
아주 큰 값 -> 1에 가까움
```

그래서 이진 분류에서 자주 사용한다.

```text
스팸이다 / 아니다
당뇨 위험 있음 / 없음
심장 질환 있음 / 없음
```

sigmoid 출력이 0.8이라면 보통 이렇게 해석할 수 있다.

```text
모델은 이 샘플이 양성일 가능성을 높게 보고 있다.
```

## tanh

tanh는 어떤 숫자를 넣어도 -1과 1 사이 값으로 바꿔준다.

```text
아주 작은 값 -> -1에 가까움
0 근처 값 -> 0 근처
아주 큰 값 -> 1에 가까움
```

tanh는 값의 중심이 0이다. 그래서 양수 방향과 음수 방향을 균형 있게 표현할 때 도움이 된다.

RNN이나 LSTM 같은 시계열 모델 내부에서 자주 만난다.

## ReLU

ReLU는 가장 자주 만나는 활성화 함수 중 하나다.

동작은 매우 단순하다.

```text
음수는 0으로 만든다.
양수는 그대로 통과시킨다.
```

예를 들어:

```text
입력: -3 -> 출력: 0
입력: 0 -> 출력: 0
입력: 5 -> 출력: 5
```

ReLU는 CNN이나 MLP의 은닉층에서 자주 사용한다. 이미지에서 의미 있는 특징은 통과시키고, 불필요한 신호는 0으로 만드는 느낌으로 이해하면 좋다.

## softmax

softmax는 여러 개의 숫자를 확률 분포처럼 바꿔준다.

예를 들어 꽃 사진을 5종류 중 하나로 분류한다고 하자.

```text
daisy
dandelion
roses
sunflowers
tulips
```

모델은 각 클래스에 대한 점수를 만들 수 있다. softmax는 이 점수들을 0과 1 사이 값으로 바꾸고, 전체 합이 1이 되게 만든다.

예:

```text
daisy: 0.05
dandelion: 0.10
roses: 0.70
sunflowers: 0.05
tulips: 0.10
```

이 경우 모델은 `roses`일 가능성을 가장 높게 본다.

## 언제 무엇을 쓰는가

처음에는 다음 표 정도만 기억하면 충분하다.

```text
은닉층 기본 선택: ReLU
이진 분류 출력층: sigmoid
다중 분류 출력층: softmax
RNN/LSTM 내부에서 자주 만남: tanh
```

## PDF 자료의 주의점

PDF 15쪽 `2.6 활성화 함수 비교 (2/2)`에는 세 번째 열 제목이 `Softmax`로 표기되어 있지만, 내용상 `ReLU`가 맞다.

근거는 다음과 같다.

```text
출력범위가 0 ~ 무한대
음수는 0으로 버리고 양수만 통과
이미지 특징 추출 예시
하단 주석의 ReLU 설명
```

따라서 그 표의 세 번째 열은 Softmax가 아니라 ReLU로 기억한다.

## 이번 장에서 반드시 기억할 문장

```text
활성화 함수는 모델의 숫자를 문제 목적에 맞는 형태로 바꿔주는 장치다.
```

그리고 이것도 기억하자.

```text
이진 분류는 sigmoid, 다중 분류는 softmax, 은닉층은 ReLU를 먼저 떠올린다.
```
