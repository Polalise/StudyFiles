# 28. 경제 시계열 예측 프로젝트 포트폴리오 문서 만들기

## 이번 장의 목표

25장부터 27장까지 경제 시계열 데이터를 수집하고, 다음 달 CPI 예측용 데이터셋을 만들고, MLP와 LSTM을 비교했다.

이번 장의 목표는 다음 한 문장이다.

> 경제 시계열 예측 프로젝트를 포트폴리오에서 설명 가능한 문서로 정리한다.

경제 프로젝트에서는 모델보다 더 중요한 것이 있다.

```text
데이터 출처
API 키 관리
예측 문제 정의
미래 정보 누출 방지
비교 실험
한계와 해석
```

## 프로젝트 제목

제목은 데이터와 목표가 보이게 적는다.

예시:

```text
FRED 경제 지표 기반 다음 달 CPI 예측 프로젝트
```

조금 더 넓게 쓰면 다음도 가능하다.

```text
API 기반 경제 시계열 예측 딥러닝 프로젝트
```

## 문제 정의

문제 정의는 다음처럼 쓸 수 있다.

```text
FRED API로 수집한 경제 지표를 사용해 다음 달 CPI를 예측하는 시계열 회귀 프로젝트를 구축한다.
CPI, 실업률, 연방기금금리 등 월별 경제 지표를 입력으로 사용하고,
lag feature 기반 MLP와 window 기반 LSTM을 비교한다.
```

여기서 중요한 단어는 다음과 같다.

```text
API 수집
경제 시계열
다음 달 CPI
회귀
MLP/LSTM 비교
```

## 데이터 수집 설명

경제 데이터는 FRED API를 통해 수집했다.

포트폴리오에서는 키 값을 공개하면 안 된다.

좋은 설명은 다음과 같다.

```text
FRED API를 사용해 CPI, 실업률, 연방기금금리 데이터를 수집했다.
API 키는 .env 파일로 관리해 코드에 직접 노출하지 않았다.
API 응답 JSON을 pandas DataFrame으로 변환하고, 날짜와 숫자 타입을 정리한 뒤 CSV로 저장했다.
```

## 데이터셋 설계 설명

경제 예측에서 중요한 것은 target 설계다.

이번 프로젝트에서는 다음 달 CPI를 정답으로 만들었다.

```python
target_next_cpi = cpi.shift(-1)
```

이 말은 다음과 같다.

```text
현재 달의 경제 지표를 보고 다음 달 CPI를 예측한다.
```

입력 특성에는 현재 값과 과거 lag 값을 사용했다.

```text
cpi_lag_1
cpi_lag_2
cpi_lag_3
unemployment_rate_lag_1
fed_funds_rate_lag_1
...
```

## 미래 정보 누출 방지

경제 시계열에서는 미래 정보 누출을 매우 조심해야 한다.

이번 프로젝트에서 지킨 원칙은 다음과 같다.

```text
입력 X에는 현재와 과거 값만 사용한다.
정답 y는 다음 달 CPI로 만든다.
train/validation split은 시간 순서대로 나눈다.
scaler는 train 데이터에만 fit한다.
validation 데이터에는 transform만 적용한다.
```

이 부분은 포트폴리오에서 꼭 강조할 만하다.

## 모델 비교 설명

이번 프로젝트에서는 두 가지 접근을 비교했다.

```text
MLP  = lag feature를 2차원 표 형태 입력으로 사용
LSTM = 과거 6개월의 경제 지표 흐름을 3차원 입력으로 사용
```

MLP 입력은 다음처럼 해석한다.

```text
(샘플 수, 특성 수)
```

LSTM 입력은 다음처럼 해석한다.

```text
(샘플 수, 6개월, 경제 지표 수)
```

중요한 것은 “LSTM을 썼다”가 아니라, 두 방식을 같은 기준으로 비교했다는 점이다.

## 평가 지표

다음 달 CPI 예측은 회귀 문제다.

그래서 평가 지표는 MAE와 RMSE를 사용했다.

```text
MAE  = 평균적으로 CPI를 얼마나 틀렸는가
RMSE = 큰 오차에 더 민감한 오차 지표
```

경제 데이터에서는 예측값이 아주 정확하다고 과장하면 안 된다.

경제 지표는 정책 변화, 위기, 외부 충격에 의해 갑자기 변할 수 있기 때문이다.

## 한계

경제 시계열 프로젝트의 한계는 명확히 적는 것이 좋다.

```text
사용한 지표 수가 제한적이다.
월별 데이터라 샘플 수가 많지 않다.
경제 데이터는 나중에 수정될 수 있다.
정책 변화나 외부 충격을 모델이 충분히 반영하지 못할 수 있다.
LSTM이 항상 MLP보다 좋은 것은 아니다.
예측 결과를 투자 판단이나 정책 판단으로 사용하면 안 된다.
```
