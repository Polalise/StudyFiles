# 21-a. RNN, LSTM, GRU, BiLSTM 비교

## 왜 21장 뒤에 이 장을 배우는가

21장에서는 PM10 시계열 데이터를 LSTM 모델에 넣어 다음 시간의 PM10 값을 예측했다.

그때 가장 중요한 입력 모양은 이것이었다.

```text
(samples, timesteps, features)
```

예를 들어 과거 24시간의 PM10 하나만 보고 다음 값을 예측한다면 입력 모양은 다음과 같다.

```text
(샘플 수, 24, 1)
```

21장에서는 이 입력을 LSTM에 넣었다.
하지만 시간 순서 데이터를 다루는 층은 LSTM만 있는 것이 아니다.

이번 장에서는 같은 입력 데이터에 여러 순환 모델을 넣어 비교한다.

```text
SimpleRNN
LSTM
GRU
BiLSTM
```

핵심은 새로운 전처리를 배우는 것이 아니다.
이미 만든 window 데이터를 그대로 두고, 모델 층만 바꿔보는 것이다.

## SimpleRNN은 무엇인가

SimpleRNN은 가장 기본적인 순환 신경망이다.

순환이라는 말은 이전 시간의 정보를 다음 시간으로 넘긴다는 뜻이다.

```text
1시간 전 정보 -> 2시간 전 정보와 함께 사용
2시간 전 정보 -> 3시간 전 정보와 함께 사용
...
```

비유하면 짧은 메모장을 들고 시간 순서대로 데이터를 읽는 모델이다.

하지만 SimpleRNN은 긴 흐름을 오래 기억하는 데 약하다.
과거 정보가 길어질수록 앞쪽 정보가 희미해질 수 있다.

그래서 긴 시계열에서는 LSTM이나 GRU가 자주 사용된다.

## LSTM은 무엇인가

LSTM은 Long Short-Term Memory의 줄임말이다.

이름 그대로 짧은 정보와 긴 정보를 더 잘 다루기 위해 만들어진 순환 모델이다.

LSTM은 내부에 정보를 얼마나 기억하고, 얼마나 버릴지 조절하는 장치를 가진다.
이 장치를 보통 gate라고 부른다.

처음에는 gate의 수식을 외울 필요는 없다.
다만 이렇게 이해하면 된다.

```text
SimpleRNN보다 오래 기억하기 위한 장치가 있는 모델
```

21장에서 LSTM을 먼저 배운 이유도 이 때문이다.
시계열 예측에서 기본 선택지로 자주 등장한다.

## GRU는 무엇인가

GRU는 Gated Recurrent Unit의 줄임말이다.

GRU도 LSTM처럼 gate를 사용한다.
하지만 구조가 LSTM보다 조금 단순하다.

그래서 이런 장점이 있을 수 있다.

```text
학습이 조금 빠를 수 있다.
파라미터 수가 더 적을 수 있다.
데이터나 문제에 따라 LSTM과 비슷한 성능을 낼 수 있다.
```

항상 GRU가 더 좋거나 LSTM이 더 좋은 것은 아니다.
실제로 비교해보는 것이 중요하다.

## BiLSTM은 무엇인가

BiLSTM은 Bidirectional LSTM의 줄임말이다.
양방향 LSTM이라고 부른다.

일반 LSTM은 시간 순서를 한 방향으로 읽는다.

```text
과거 -> 현재
```

BiLSTM은 두 방향으로 읽는다.

```text
과거 -> 현재
현재 -> 과거
```

문장 분류처럼 전체 문장을 이미 알고 있는 문제에서는 BiLSTM이 도움이 될 수 있다.
예를 들어 문장 감정 분석에서는 문장의 앞과 뒤를 모두 보고 판단해도 된다.

하지만 시계열 예측에서는 조심해야 한다.
미래 값을 예측하는 문제에서 실제 미래 정보를 모델이 보게 만들면 데이터 누출이 된다.

이번 실습의 BiLSTM은 각 window 내부의 24시간을 양방향으로 읽는다.
즉 입력 window 안에서만 양방향을 쓰는 구조다.
그래도 실제 예측 시스템에서는 BiLSTM 사용 이유와 데이터 누출 가능성을 반드시 설명해야 한다.

초보 단계에서는 이렇게 기억하자.

```text
BiLSTM은 강력할 수 있지만, 미래 예측 문제에서는 조심해서 써야 한다.
```

## 비교할 때 주의할 점

모델을 비교할 때는 조건을 최대한 맞춰야 한다.

```text
같은 데이터
같은 train/test split
같은 window size
같은 epoch
같은 batch size
같은 평가 지표
```

그래야 모델 종류의 차이를 조금 더 공정하게 볼 수 있다.

이번 장에서는 빠른 실행을 위해 epoch를 작게 둔다.
따라서 결과 숫자 하나로 최종 결론을 내리면 안 된다.

이번 실습의 목적은 다음이다.

```text
순환 모델의 종류를 바꾸어 끼우는 방법을 익힌다.
결과 표를 만들고 비교하는 흐름을 배운다.
```

## 이번 장에서 반드시 기억할 문장

```text
SimpleRNN, LSTM, GRU, BiLSTM은 모두 시간 순서 데이터를 다룰 수 있지만, 기억 방식과 사용상 주의점이 다르다.
```

하나 더 기억하자.

```text
시계열 예측에서는 모델 성능뿐 아니라 데이터 누출이 없는 비교인지 반드시 확인해야 한다.
```
