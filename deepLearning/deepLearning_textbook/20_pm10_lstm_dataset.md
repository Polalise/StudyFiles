# 20. PM10 데이터를 LSTM 입력 모양으로 만들기

## 이번 장의 목표

19장에서는 PM10 데이터를 읽고 시간 흐름을 확인했다.

이번 장에서는 LSTM에 넣을 수 있는 모양으로 데이터를 바꾼다.

목표는 다음 한 문장이다.

> 과거 여러 시간의 PM10 값을 입력 X로 만들고, 다음 시간의 PM10 값을 정답 y로 만든다.

시계열 딥러닝에서 가장 중요한 문장이다.

```text
과거를 보고 미래를 맞힌다.
```

## LSTM은 무엇을 기대할까

일반 MLP는 보통 이런 입력을 받는다.

```text
(샘플 수, 특성 수)
```

예를 들어 건강 데이터에서는 한 사람이 하나의 샘플이고, BMI, 나이, 혈압 같은 특성이 있었다.

하지만 LSTM은 시간 순서를 보기 때문에 입력 모양이 다르다.

```text
(샘플 수, 시간 길이, 특성 수)
```

영어로는 이렇게 부른다.

```text
samples, timesteps, features
```

## samples, timesteps, features 읽는 법

예를 들어 입력 모양이 다음과 같다고 하자.

```text
(1000, 24, 1)
```

이것은 이렇게 읽는다.

```text
샘플이 1000개 있다.
각 샘플은 과거 24시간을 본다.
각 시간에는 특성이 1개 있다.
```

PM10만 사용하면 특성 수는 1개다.

PM10과 PM2.5를 함께 사용하면 특성 수는 2개가 될 수 있다.

이번 장에서는 먼저 PM10 하나만 사용한다.

## window란 무엇인가

시계열에서 window는 과거 몇 개 값을 한 묶음으로 볼 것인지 정하는 길이다.

예를 들어 window가 3이라고 하자.

PM10 값이 다음처럼 있다.

```text
10, 20, 30, 40, 50, 60
```

그러면 입력과 정답은 이렇게 만들 수 있다.

```text
X = [10, 20, 30] -> y = 40
X = [20, 30, 40] -> y = 50
X = [30, 40, 50] -> y = 60
```

즉, 과거 3개 값을 보고 다음 1개 값을 맞힌다.

PM10 시간별 데이터에서 window를 24로 두면 이런 뜻이다.

```text
과거 24시간의 PM10을 보고 다음 시간 PM10을 맞힌다.
```

## train/validation split은 시간 순서대로

시계열에서는 무작위로 섞어서 나누면 안 되는 경우가 많다.

미래 데이터가 학습에 섞일 수 있기 때문이다.

그래서 이번 장에서는 앞쪽 80%를 학습용, 뒤쪽 20%를 검증용으로 사용한다.

```text
과거 80% -> 학습
미래 20% -> 검증
```

이 방식이 더 현실적인 예측 상황에 가깝다.

## 스케일링도 시간 순서를 지킨다

시계열에서도 scaler는 학습 데이터 기준으로만 fit해야 한다.

```text
train 데이터로 scaler.fit
train 데이터 transform
validation 데이터 transform
```

검증 데이터의 평균과 표준편차를 미리 써버리면 미래 정보를 몰래 본 셈이 될 수 있다.

## 이번 장의 실습 흐름

```text
PM10 데이터 읽기
-> 강남구 선택
-> 결측치 처리
-> 시간 순서 정렬
-> train/validation 시간 순서 분리
-> MinMaxScaler 적용
-> window 데이터셋 생성
-> LSTM 입력 shape 확인
```

이번 장에서는 아직 모델을 본격적으로 학습하지 않는다.

먼저 LSTM이 받을 입력 모양을 확실히 이해한다.

## 이번 장의 과제

1. `(1000, 24, 1)` 모양을 자기 말로 설명해보자.
2. window size가 24라는 말이 PM10 예측에서 무엇을 의미하는지 설명해보자.
3. 시계열에서 무작위 split을 피해야 하는 이유를 설명해보자.
4. scaler를 train 데이터에만 fit해야 하는 이유를 설명해보자.
