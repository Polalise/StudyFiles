# 21. LSTM으로 PM10 다음 시간 예측하기

## 이번 장의 목표

20장에서는 PM10 데이터를 LSTM 입력 모양으로 만들었다.

이번 장에서는 그 데이터를 실제 LSTM 모델에 넣어 다음 시간의 PM10 값을 예측한다.

목표는 다음 한 문장이다.

> 과거 24시간의 PM10 값을 보고 다음 1시간의 PM10 값을 예측하는 LSTM 모델을 만든다.

이번 문제는 분류가 아니라 회귀다.

```text
분류 = 정답이 종류
회귀 = 정답이 숫자
```

PM10 예측은 다음 시간의 농도 숫자를 맞히는 문제이므로 회귀다.

## LSTM은 왜 쓰는가

MLP는 입력값을 한 덩어리 숫자 묶음으로 본다.

하지만 LSTM은 시간 순서를 고려한다.

예를 들어 과거 24시간 PM10 값이 있다면, LSTM은 다음 흐름을 본다.

```text
1시간 전 값
2시간 전 값
...
24시간 전 값
```

물론 LSTM이 마법처럼 미래를 정확히 맞히는 것은 아니다. 다만 시간 순서가 있는 패턴을 다루기 위해 설계된 층이다.

## 모델 구조

이번 장의 기본 모델은 단순하게 시작한다.

```text
입력: 과거 24시간 PM10
-> LSTM
-> Dense
-> 출력: 다음 시간 PM10
```

Keras 코드로는 대략 이런 모양이다.

```python
model = Sequential([
    Input(shape=(24, 1)),
    LSTM(32),
    Dense(1)
])
```

여기서 `Input(shape=(24, 1))`은 이렇게 읽는다.

```text
각 샘플은 24개 시간 단계가 있고,
각 시간 단계에는 특성 1개가 있다.
```

## 출력층에 activation이 없는 이유

분류에서는 출력층에 sigmoid나 softmax를 자주 썼다.

하지만 이번 문제는 PM10 숫자를 예측하는 회귀 문제다.

PM10 값은 0과 1 사이 확률이 아니다. 그래서 출력층은 보통 activation 없이 둔다.

```python
Dense(1)
```

이 말은 숫자 하나를 그대로 출력한다는 뜻이다.

## 손실 함수는 왜 mse인가

회귀 문제에서는 평균제곱오차, 즉 MSE를 자주 사용한다.

```python
loss="mse"
```

MSE는 예측값과 실제값의 차이를 제곱해서 평균낸 값이다.

처음에는 이렇게 이해하면 충분하다.

```text
예측이 정답과 멀수록 손실이 커진다.
큰 오차는 더 크게 벌점을 받는다.
```

## MAE와 RMSE

모델 평가에서는 MAE와 RMSE를 함께 본다.

```text
MAE  = 평균적으로 몇 PM10 정도 틀렸는가
RMSE = 큰 오차에 더 민감한 평균 오차
```

예를 들어 MAE가 8이라면 대략 이렇게 말할 수 있다.

```text
검증 구간에서 평균적으로 약 8 PM10 정도 차이가 났다.
```

정확도처럼 0~1 사이 점수가 아니라, 원래 PM10 단위로 해석하는 것이 중요하다.

## inverse_transform이 중요한 이유

20장에서 PM10 값을 `MinMaxScaler`로 0~1 범위로 바꿨다.

모델은 스케일링된 값을 예측한다.

하지만 사람은 0.35 같은 스케일링 값보다 실제 PM10 단위를 보고 싶다.

그래서 예측값과 정답을 다시 원래 단위로 되돌린다.

```python
scaler.inverse_transform(...)
```

## 이번 장의 실습 흐름

```text
PM10 데이터 읽기
-> window 데이터셋 생성
-> LSTM 모델 만들기
-> 학습
-> 예측
-> 원래 PM10 단위로 복원
-> MAE/RMSE 계산
-> 실제값과 예측값 그래프 비교
```

## 이번 장의 과제

1. PM10 예측이 왜 분류가 아니라 회귀인지 설명해보자.
2. `Input(shape=(24, 1))`을 자기 말로 읽어보자.
3. `Dense(1)`에 sigmoid를 쓰지 않는 이유를 설명해보자.
4. MAE가 8이라는 말이 무엇을 뜻하는지 설명해보자.
