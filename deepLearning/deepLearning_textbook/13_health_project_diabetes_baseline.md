# 13. 건강 데이터 프로젝트 시작: 당뇨병 위험 예측 Baseline

## 이번 장의 목표

이미지 파트에서는 꽃 사진을 분류하는 CNN 프로젝트를 만들었다.

이번 장부터는 표 형태의 건강 데이터를 사용한다.

이번 장의 목표는 다음 한 문장이다.

> 건강 지표 데이터를 보고, 당뇨병 위험 예측 프로젝트의 문제 정의와 첫 baseline 모델을 만든다.

여기서 baseline은 “처음 기준점이 되는 모델”이라는 뜻이다.

처음부터 멋진 모델을 만들려고 하면 길을 잃기 쉽다. 그래서 먼저 단순하지만 제대로 된 기준 모델을 만든다.

## 사용할 데이터

이번 장에서는 다음 파일을 사용한다.

```text
C:\work\dataset\diabetes_or_cardiovascular\diabetes_binary_5050split_health_indicators_BRFSS2015.csv
```

이 데이터는 건강 지표를 바탕으로 당뇨병 여부를 예측하는 이진 분류 데이터다.

타깃 컬럼은 다음과 같다.

```text
Diabetes_binary
```

값의 의미는 다음처럼 이해하면 된다.

```text
0 = 당뇨병 없음
1 = 당뇨병 있음
```

중요한 표현 주의:

```text
이 모델은 의학적 진단 도구가 아니다.
교육용 건강 위험 예측 예제다.
```

포트폴리오에서도 반드시 이렇게 표현해야 한다. “진단한다”가 아니라 “위험을 예측한다”, “분류 모델을 실험했다”라고 말하는 것이 안전하다.

## 건강 데이터 프로젝트에서 가장 먼저 볼 것

딥러닝 모델을 만들기 전에 반드시 다음을 확인해야 한다.

```text
1. 무엇을 예측하는가?
2. 입력으로 무엇을 사용하는가?
3. 정답이 0/1인지 여러 개인지 확인한다.
4. 데이터가 한쪽 클래스로 치우쳐 있는지 확인한다.
5. 누출될 수 있는 정보가 있는지 확인한다.
6. 어떤 평가 지표를 볼지 정한다.
```

초보자는 모델 코드부터 쓰고 싶어지지만, 실제 프로젝트에서는 문제 정의가 먼저다.

## 입력 컬럼 이해하기

이 데이터에는 혈압, 콜레스테롤, BMI, 흡연 여부, 신체활동, 나이 구간, 소득 구간 같은 건강 관련 지표가 들어 있다.

예를 들면 다음과 같다.

```text
HighBP      = 고혈압 여부
HighChol    = 고콜레스테롤 여부
BMI         = 체질량지수
Smoker      = 흡연 경험 여부
Stroke      = 뇌졸중 경험 여부
PhysActivity = 신체활동 여부
GenHlth     = 스스로 평가한 전반적 건강 상태
Age         = 나이 구간
Income      = 소득 구간
```

컬럼 이름이 영어라 처음에는 낯설 수 있다. 하지만 핵심은 간단하다.

```text
한 행 = 한 사람의 건강 설문/지표
한 열 = 그 사람을 설명하는 특징
타깃 = 당뇨병 여부
```

## 왜 baseline이 필요한가

baseline은 프로젝트의 출발선이다.

처음 만든 baseline이 있어야 나중에 이런 비교가 가능하다.

```text
전처리를 바꾸니 좋아졌는가?
MLP 구조를 바꾸니 좋아졌는가?
Dropout을 넣으니 과적합이 줄었는가?
평가 지표가 실제로 개선되었는가?
```

baseline이 없으면 개선인지 아닌지 알 수 없다.

## 이번 장의 모델

이번 장에서는 아주 단순한 MLP를 사용한다.

```text
입력 건강 지표
-> Dense
-> Dropout
-> Dense
-> sigmoid
-> 당뇨병 위험 확률
```

출력층은 다음처럼 만든다.

```python
Dense(1, activation="sigmoid")
```

sigmoid는 숫자를 0과 1 사이로 바꾼다.

이진 분류에서는 이 값을 “1일 가능성”처럼 해석한다.

```text
0.12 = 당뇨병 있음으로 볼 가능성이 낮음
0.83 = 당뇨병 있음으로 볼 가능성이 높음
```

단, 이것은 의학적 확정 진단이 아니다. 모델이 데이터 패턴을 보고 낸 예측 점수다.

## 손실 함수

이번 문제는 이진 분류다.

그래서 손실 함수는 다음을 사용한다.

```python
loss="binary_crossentropy"
```

읽는 법은 이렇게 잡으면 된다.

```text
binary = 정답이 0 또는 1인 문제
crossentropy = 예측 확률과 실제 정답의 차이를 재는 분류용 손실
```

## 평가 지표

정확도만 보면 부족할 수 있다.

건강 위험 예측에서는 특히 다음 지표를 함께 봐야 한다.

```text
accuracy  = 전체 중 맞힌 비율
precision = 위험이라고 예측한 사람 중 실제 위험인 비율
recall    = 실제 위험인 사람 중 모델이 찾아낸 비율
f1-score  = precision과 recall을 함께 본 점수
```

건강 데이터에서는 recall이 중요해지는 경우가 많다.

왜냐하면 실제 위험이 있는 사람을 놓치는 것이 큰 문제가 될 수 있기 때문이다.

하지만 recall만 높이면 아무나 위험하다고 예측할 수도 있다. 그래서 precision과 함께 봐야 한다.
