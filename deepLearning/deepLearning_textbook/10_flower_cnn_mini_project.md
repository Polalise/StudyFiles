# 10. 꽃 이미지 CNN 미니 프로젝트

## 이번 장의 목표

9장에서는 손글씨 숫자처럼 이미 정리된 이미지 데이터로 CNN을 배웠다. 이번 장에서는 실제 폴더에 들어 있는 꽃 사진을 사용한다.

이번 장의 목표는 다음 한 문장이다.

> 폴더에 나뉘어 있는 이미지 파일을 Keras가 읽을 수 있는 데이터셋으로 바꾸고, CNN으로 꽃 종류를 분류하는 작은 프로젝트를 완성한다.

사용할 데이터는 다음 위치에 있다.

```text
C:\work\deepLearning\flower_dataset\flower_photos
```

폴더 구조는 대략 다음과 같다.

```text
flower_photos
├─ daisy
├─ dandelion
├─ roses
├─ sunflowers
└─ tulips
```

여기서 중요한 점은 “폴더 이름이 정답 이름이 된다”는 것이다.

예를 들어 `roses` 폴더 안에 있는 이미지는 정답이 `roses`이고, `tulips` 폴더 안에 있는 이미지는 정답이 `tulips`이다.

## 왜 이번 장이 프로젝트에 가까운가

딥러닝 예제에는 두 종류가 있다.

첫 번째는 학습용 예제다. MNIST처럼 데이터가 이미 숫자 배열로 깔끔하게 준비되어 있다. 처음 배울 때는 좋지만, 실제 프로젝트와는 조금 거리가 있다.

두 번째는 실제 파일 기반 예제다. 이미지 파일이 폴더에 있고, 그 폴더를 읽고, 크기를 맞추고, 학습용과 검증용으로 나누고, 모델을 저장한다. 실제 프로젝트는 대부분 이쪽에 가깝다.

이번 장은 두 번째 방식이다.

## 핵심 흐름

이미지 분류 프로젝트의 기본 흐름은 다음과 같다.

```text
이미지 폴더 확인
-> 이미지 크기 통일
-> 학습 데이터와 검증 데이터 분리
-> 픽셀 값을 0~1 사이로 조정
-> CNN 모델 만들기
-> 학습
-> 검증 정확도 확인
-> 새 이미지 예측
-> 모델 저장
```

이 흐름은 꽃 분류뿐 아니라 음식 분류, 불량품 분류, 피부 이미지 분류, 표정 분류 같은 프로젝트에도 거의 그대로 쓰인다.

## 이미지 크기를 왜 맞출까

사진 파일은 크기가 제각각이다.

어떤 사진은 `640 x 480`일 수 있고, 어떤 사진은 `1024 x 768`일 수 있다. 사람은 크기가 달라도 꽃을 알아보지만, 신경망은 입력 모양이 일정해야 한다.

그래서 모든 이미지를 같은 크기로 바꾼다.

이번 장에서는 CPU 환경을 고려해서 `160 x 160` 크기를 사용한다.

```text
원본 이미지들: 크기가 제각각
모델 입력: 160 x 160 x 3
```

여기서 마지막 `3`은 색상 채널이다.

```text
R = 빨강
G = 초록
B = 파랑
```

즉, 컬러 이미지는 보통 `높이 x 너비 x 3` 모양이다.

## 배치란 무엇인가

배치(batch)는 이미지를 몇 장씩 묶어서 모델에게 보여줄지 정하는 단위다.

예를 들어 `batch_size = 32`라면, 모델은 한 번에 이미지 32장을 보고 가중치를 조금씩 수정한다.

배치를 식당 비유로 보면 이렇다.

```text
이미지 1장씩 조리하기: 너무 자주 왔다 갔다 해서 비효율적
이미지 전체를 한 번에 조리하기: 너무 커서 부담됨
이미지 32장씩 묶어 조리하기: 적당히 효율적
```

이번 장에서는 CPU 환경을 고려해서 너무 큰 배치를 쓰지 않는다.

## 데이터 증강이란 무엇인가

이미지 분류에서는 데이터 증강(data augmentation)을 자주 쓴다.

데이터 증강은 원본 이미지를 약간 바꾸어 모델에게 보여주는 방법이다.

예를 들어 다음과 같은 변화가 가능하다.

```text
좌우 뒤집기
조금 회전하기
조금 확대하기
```

이렇게 하면 모델이 “꽃이 반드시 정중앙에 똑바로 있어야 한다”고 외우지 않게 된다.

즉, 데이터 증강은 모델에게 이런 태도를 가르친다.

> 꽃이 조금 돌아가 있거나 위치가 달라도 같은 꽃일 수 있다.

## 이번 장의 모델 구조

이번 장의 CNN은 다음 흐름을 가진다.

```text
이미지 입력
-> 데이터 증강
-> 픽셀값 0~1 변환
-> Conv2D
-> MaxPooling2D
-> Conv2D
-> MaxPooling2D
-> Conv2D
-> MaxPooling2D
-> Flatten
-> Dense
-> Dropout
-> Dense
-> 꽃 종류 예측
```

`Conv2D`는 이미지에서 작은 특징을 찾는 층이다.

초반 층은 선, 색 변화, 가장자리 같은 단순한 특징을 찾는다. 뒤쪽 층은 꽃잎 모양, 중심부 패턴, 전체적인 색 조합처럼 더 복잡한 특징을 조합한다.

`MaxPooling2D`는 중요한 특징만 남기면서 이미지 크기를 줄인다.

`Dropout`은 학습 중 일부 연결을 잠시 꺼서, 모델이 특정 특징에만 지나치게 의존하지 않도록 돕는다.

## 손실 함수는 왜 SparseCategoricalCrossentropy인가

이번 문제는 다중 분류다.

정답은 다음 5개 중 하나다.

```text
daisy
dandelion
roses
sunflowers
tulips
```

Keras의 `image_dataset_from_directory`는 정답을 보통 숫자로 만든다.

```text
daisy      -> 0
dandelion  -> 1
roses      -> 2
sunflowers -> 3
tulips     -> 4
```

정답이 이런 정수 번호 형태일 때는 `SparseCategoricalCrossentropy`를 사용하면 된다.

읽는 법은 이렇게 잡으면 된다.

```text
Sparse = 정답이 원-핫 벡터가 아니라 정수 번호다
Categorical = 여러 클래스 중 하나를 고르는 문제다
Crossentropy = 분류 문제에서 예측 확률과 정답의 차이를 재는 손실이다
```

## 이번 장에서 조심할 점

첫째, 이 데이터는 이미지 수가 아주 크지는 않다. 그래서 학습 정확도는 올라가는데 검증 정확도는 덜 올라갈 수 있다. 이것은 과적합일 수 있다.

둘째, 현재 컴퓨터는 강한 딥러닝 GPU 환경이 아니다. 그래서 모델을 너무 크게 만들면 학습이 느릴 수 있다. 이번 장에서는 작은 CNN으로 시작한다.

셋째, 이 모델은 교육용이다. 포트폴리오로 발전시키려면 결과 해석, 오답 이미지 확인, 모델 개선 실험, 간단한 데모 화면까지 붙이는 것이 좋다.

## 이번 장의 과제

1. `IMG_SIZE`를 `128`, `160`, `180` 중 하나로 바꾸면 어떤 장단점이 있을지 글로 정리해보자.
2. `Dropout(0.3)` 값을 `0.1`, `0.5`로 바꾸면 어떤 변화가 예상되는지 적어보자.
3. 학습 결과에서 `accuracy`와 `val_accuracy`의 차이가 커진다면 어떤 의미인지 설명해보자.
4. 모델이 틀린 이미지를 직접 확인하는 코드를 다음 장에서 만들 예정이므로, 이번 장에서는 “틀린 예측을 왜 봐야 하는가”를 자기 말로 적어보자.
