# 01. 딥러닝의 기본 리듬

## 이번 장에서 배울 것

이번 장의 목표는 딥러닝을 어려운 공식이 아니라 하나의 반복 흐름으로 이해하는 것이다.

딥러닝의 가장 중요한 리듬은 다음 네 단어다.

```text
입력 -> 예측 -> 손실 -> 수정
```

이 네 단어를 이해하면, 앞으로 나올 CNN, LSTM, GRU, GAN 같은 이름이 조금 덜 무서워진다. 모델 종류는 달라도 기본 리듬은 계속 반복되기 때문이다.

## 딥러닝을 아주 쉬운 말로 말하면

딥러닝은 컴퓨터가 데이터를 보고 규칙을 찾아가는 방법이다.

사람은 다음 숫자를 보면 금방 규칙을 눈치챌 수 있다.

```text
x: 0, 1, 2, 3, 4
y: 1, 3, 5, 7, 9
```

사람은 이렇게 생각할 수 있다.

```text
x에 2를 곱하고 1을 더하면 y가 된다.
```

즉 규칙은 다음과 같다.

```text
y = 2x + 1
```

하지만 컴퓨터 모델은 처음부터 이 규칙을 모른다. 처음에는 엉뚱한 예측을 한다. 그리고 정답과 비교하면서 조금씩 내부 숫자를 고친다.

## 모델은 무엇을 배우는가

이번 장에서는 가장 단순한 모델을 사용한다.

```text
y = wx + b
```

이 수식은 이렇게 읽으면 된다.

```text
예측값은 입력값에 가중치를 곱하고 편향을 더해서 만든다.
```

기호별 뜻은 다음과 같다.

```text
x: 입력값
y: 모델이 만든 예측값
w: 가중치, 입력값에 얼마나 힘을 줄지 정하는 숫자
b: 편향, 기본으로 더해지는 숫자
```

우리가 원하는 정답 규칙은 다음과 같다.

```text
y = 2x + 1
```

그러면 모델이 최종적으로 찾아야 하는 값은 다음과 같다.

```text
w = 2
b = 1
```

처음에는 모델이 `w`와 `b`를 모른다. 딥러닝 학습은 이 두 숫자를 정답에 가까워지도록 고치는 과정이라고 볼 수 있다.

## 딥러닝의 기본 흐름

모델 학습은 다음 흐름으로 진행된다.

```text
1. 입력을 넣는다.
2. 모델이 예측한다.
3. 예측값과 정답을 비교한다.
4. 얼마나 틀렸는지 손실(loss)을 계산한다.
5. 손실이 줄어드는 방향으로 w와 b를 조금 수정한다.
6. 이 과정을 여러 번 반복한다.
```

그림처럼 보면 다음과 같다.

```text
        입력 x
          |
          v
   [ 모델 y = wx + b ]
          |
          v
        예측값
          |
          v
   정답과 비교해서 손실 계산
          |
          v
   w와 b를 조금 수정
          |
          v
        다시 반복
```

## 손실이란 무엇인가

손실은 모델이 얼마나 틀렸는지를 숫자로 나타낸 것이다.

```text
손실이 크다 = 많이 틀렸다
손실이 작다 = 조금 틀렸다
손실이 0에 가깝다 = 거의 맞았다
```

비유하자면 손실은 목적지까지 남은 거리와 비슷하다.

```text
목적지까지 100m 남음 -> 아직 멀다
목적지까지 10m 남음 -> 가까워졌다
목적지까지 0m 남음 -> 도착했다
```

모델은 손실을 줄이는 방향으로 내부 숫자인 `w`와 `b`를 고친다.

## 학습은 한 번에 끝나지 않는다

사람도 처음 푸는 문제를 한 번에 완벽히 맞히기 어렵다. 모델도 마찬가지다.

모델은 같은 데이터를 여러 번 보면서 조금씩 좋아진다. 이때 전체 데이터를 한 번 다 보는 것을 `epoch`라고 한다.

```text
epoch = 전체 문제집을 한 번 다 푸는 것
epochs=50 = 같은 문제집을 50번 반복해서 푸는 것
```

## 코드에서 이 개념이 어디에 나오는가

실습 노트북에서는 다음 순서로 확인한다.

```python
x_data = np.array([[0], [1], [2], [3], [4]], dtype=np.float32)
y_data = np.array([[1], [3], [5], [7], [9]], dtype=np.float32)
```

이 부분은 입력과 정답을 준비하는 단계다.

Keras 모델은 보통 입력을 `(샘플 수, 특성 수)` 형태로 받는다. 여기서는 샘플이 5개이고 각 샘플에는 `x` 하나만 있으므로 `x_data`의 모양은 `(5, 1)`이다.

```python
model = Sequential([
    Input(shape=(1,)),
    Dense(1)
])
```

이 부분은 아주 단순한 모델을 만드는 단계다. 입력 1개를 받아 출력 1개를 만든다.

```python
model.compile(optimizer="sgd", loss="mse")
```

이 부분은 모델에게 학습 방법을 알려주는 단계다.

- `optimizer`: 손실을 줄이기 위해 내부 숫자를 수정하는 방법
- `loss`: 예측이 얼마나 틀렸는지 재는 기준

```python
model.fit(x_data, y_data, epochs=100)
```

이 부분은 실제 학습이다. 모델은 `x_data`를 보고 `y_data`에 가까운 예측을 하도록 내부 숫자를 고친다.

## 이번 장에서 반드시 기억할 문장

```text
딥러닝은 입력을 보고 예측을 만들고,
정답과 비교해서 얼마나 틀렸는지 계산한 뒤,
덜 틀리도록 내부 숫자를 조금씩 고치는 과정이다.
```

이 문장은 앞으로 계속 돌아올 기준점이다.

## 자주 헷갈리는 부분

### 모델이 규칙을 외우는 것인가?

완전히 같지는 않다. 모델은 주어진 데이터에서 패턴을 찾는다. 데이터가 너무 적거나 한쪽으로 치우쳐 있으면 잘못된 패턴을 배울 수도 있다.

### 손실과 정확도는 같은가?

다르다. 손실은 모델이 얼마나 틀렸는지를 학습 중에 줄이려는 숫자다. 정확도는 분류 문제에서 몇 개를 맞혔는지 보는 지표다. 이번 장은 회귀 문제라 정확도보다 손실을 본다.

### 왜 수식이 필요한가?

수식은 긴 설명을 짧게 적은 문장이다. 지금은 수식을 외우는 것이 목표가 아니다. `y = wx + b`를 보고 “입력에 가중치를 곱하고 편향을 더한다”라고 읽을 수 있으면 충분하다.
