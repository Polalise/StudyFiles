# 25. 경제 시계열 프로젝트 시작: API로 데이터 가져오기

## 이번 장의 목표

지금까지 이미지, 건강 표 데이터, PM10 시계열 데이터를 다뤘다.

이번 장부터는 경제 시계열 프로젝트로 넘어간다.

이번 장의 목표는 다음 한 문장이다.

> API 키를 사용해 경제 지표 시계열 데이터를 가져오고, 딥러닝 프로젝트용 데이터로 정리하는 첫 흐름을 익힌다.

경제 데이터 프로젝트는 파일 하나를 읽는 것과 조금 다르다.

CSV 파일이 이미 준비된 경우도 있지만, 실제 프로젝트에서는 API로 데이터를 가져오는 경우가 많다.

## API란 무엇인가

API는 프로그램이 다른 서비스에 데이터를 요청하는 통로다.

웹사이트에 사람이 들어가서 버튼을 누르는 대신, 파이썬 코드가 정해진 주소로 요청을 보내 데이터를 받는다.

비유하면 API는 이런 느낌이다.

```text
사람이 웹페이지에서 검색하기
-> 브라우저를 열고 클릭하고 다운로드한다.

파이썬이 API로 요청하기
-> 정해진 URL에 필요한 파라미터를 붙여 데이터를 받는다.
```

## API 키란 무엇인가

API 키는 서비스를 사용할 수 있는 열쇠다.

이번 프로젝트에서는 이미 `.env`에 다음 키들이 등록되어 있다.

```text
FRED_API_KEY
ECOS_API_KEY
KOSIS_API_KEY
KPDP_API_KEY
```

교재와 노트북에서는 API 키 값을 출력하지 않는다.

키는 비밀번호처럼 다뤄야 한다.

## 이번 장에서 먼저 사용할 데이터: FRED

FRED는 미국 세인트루이스 연방준비은행에서 제공하는 경제 데이터 서비스다.

이번 장에서는 FRED API로 경제 지표를 가져오는 흐름을 먼저 연습한다.

예를 들어 다음 지표를 가져올 수 있다.

```text
CPIAUCSL = 미국 소비자물가지수
FEDFUNDS = 미국 연방기금금리
UNRATE   = 미국 실업률
```

경제 시계열 프로젝트에서는 이런 지표들을 시간 순서대로 정리한 뒤 예측 문제로 바꿀 수 있다.

## 경제 시계열은 무엇이 다른가

경제 데이터도 시계열이다.

PM10처럼 시간 순서가 있다.

```text
2020-01
2020-02
2020-03
...
```

따라서 무작위로 섞어서 학습/검증을 나누면 위험할 수 있다.

경제 시계열에서도 기본 원칙은 같다.

```text
과거 데이터로 학습
미래 데이터로 검증
```

## 경제 데이터에서 조심할 점

경제 데이터는 다음 특징이 있다.

```text
월별, 분기별, 일별처럼 주기가 다양하다.
결측치가 있을 수 있다.
나중에 수정되는 값도 있다.
단위가 서로 다르다.
외부 충격에 크게 흔들릴 수 있다.
```

예를 들어 금리는 퍼센트 단위이고, 소비자물가지수는 지수 단위다.

여러 지표를 함께 쓰려면 날짜를 맞추고, 주기를 맞추고, 스케일링도 해야 한다.

## 이번 장의 실습 흐름

이번 장에서는 경제 모델을 바로 만들지 않는다.

먼저 데이터를 가져오는 흐름을 배운다.

```text
.env 파일에서 API 키 읽기
-> FRED API 요청 URL 만들기
-> JSON 응답 받기
-> pandas DataFrame으로 변환
-> 날짜 컬럼 datetime 변환
-> 값 컬럼 숫자로 변환
-> 그래프로 확인
-> CSV로 저장
```

## 왜 저장까지 할까

API는 매번 호출할 수도 있지만, 프로젝트에서는 가져온 원본 데이터를 파일로 저장해 두는 것이 좋다.

이유는 다음과 같다.

```text
API 호출 횟수를 줄일 수 있다.
나중에 같은 데이터로 재현할 수 있다.
전처리 전 원본을 보존할 수 있다.
인터넷 연결이 없어도 이후 실습을 할 수 있다.
```

## 포트폴리오에서의 의미

경제 데이터 프로젝트에서 API를 사용했다는 것은 좋은 신호가 될 수 있다.

포트폴리오에서는 이렇게 설명할 수 있다.

```text
FRED API를 사용해 경제 시계열 지표를 수집했다.
API 키는 .env로 관리해 코드에 직접 노출하지 않았다.
수집한 JSON 응답을 pandas DataFrame으로 변환하고, 날짜와 숫자 타입을 정리한 뒤 분석용 CSV로 저장했다.
```

## 이번 장의 과제

1. API와 CSV 파일 읽기의 차이를 설명해보자.
2. API 키를 코드에 직접 쓰면 왜 위험한지 설명해보자.
3. 경제 시계열에서 시간 순서를 지켜야 하는 이유를 설명해보자.
4. FRED에서 가져오고 싶은 경제 지표를 3개 골라보자.
