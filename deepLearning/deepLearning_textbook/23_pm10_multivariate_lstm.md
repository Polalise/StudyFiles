# 23. PM10 다변량 LSTM: PM2.5 함께 사용하기

## 이번 장의 목표

22장까지는 PM10 하나만 사용해 다음 시간 PM10을 예측했다.

이번 장에서는 PM2.5도 함께 입력으로 사용한다.

목표는 다음 한 문장이다.

> 과거 24시간의 PM10과 PM2.5를 함께 보고, 다음 시간의 PM10을 예측하는 다변량 LSTM 입력을 만든다.

여기서 다변량이라는 말은 한 시점에 특성이 여러 개 있다는 뜻이다.

```text
단변량: PM10 하나만 사용
다변량: PM10, PM2.5처럼 여러 특성을 함께 사용
```

## features가 1에서 2로 바뀐다

20장과 21장에서 LSTM 입력 모양은 이런 식이었다.

```text
(samples, 24, 1)
```

이 뜻은 다음과 같다.

```text
각 샘플은 과거 24시간을 보고,
각 시간에는 PM10 특성 1개가 있다.
```

이번 장에서는 PM10과 PM2.5를 함께 사용하므로 입력 모양이 이렇게 바뀐다.

```text
(samples, 24, 2)
```

이 뜻은 다음과 같다.

```text
각 샘플은 과거 24시간을 보고,
각 시간에는 PM10과 PM2.5 특성 2개가 있다.
```

## 정답은 여전히 PM10 하나

입력 특성은 2개가 되었지만, 정답은 여전히 다음 시간 PM10 하나다.

```text
X = 과거 24시간의 PM10, PM2.5
y = 다음 시간 PM10
```

따라서 출력층은 여전히 `Dense(1)`이다.

## 왜 PM2.5를 같이 볼까

PM10과 PM2.5는 둘 다 대기질과 관련된 미세먼지 지표다.

항상 같은 방식으로 움직인다고 단정할 수는 없지만, 서로 관련이 있을 가능성이 있다.

모델에게 PM10만 보여주는 것보다 PM2.5도 함께 보여주면 다음 PM10 예측에 도움이 될 수 있다.

하지만 실제로 도움이 되는지는 실험으로 확인해야 한다.

```text
PM10 단변량 LSTM
PM10 + PM2.5 다변량 LSTM
```

이 둘을 같은 기준으로 비교해야 한다.

## 다변량 스케일링

PM10과 PM2.5는 둘 다 숫자지만 범위가 다를 수 있다.

그래서 `MinMaxScaler`로 두 컬럼을 함께 스케일링한다.

```text
입력 컬럼: pm10_filled, pm25_filled
스케일링 결과: 두 컬럼이 0~1 근처 범위로 변환
```

중요한 규칙은 그대로다.

```text
scaler는 train 데이터에만 fit한다.
validation 데이터에는 transform만 한다.
```

## window 만들기

단변량에서는 window 하나가 이런 모양이었다.

```text
[
  [pm10],
  [pm10],
  [pm10],
  ...
]
```

다변량에서는 이렇게 된다.

```text
[
  [pm10, pm2.5],
  [pm10, pm2.5],
  [pm10, pm2.5],
  ...
]
```

즉, 시간축은 그대로 있고, 각 시간에 들어 있는 특성 수가 늘어난다.

## 이번 장의 실습 흐름

```text
PM10/PM2.5 데이터 읽기
-> 결측치 처리
-> PM10, PM2.5 두 컬럼 선택
-> 시간 순서 train/validation split
-> 다변량 MinMaxScaler 적용
-> window 데이터셋 생성
-> LSTM 입력 shape 확인
-> 다변량 LSTM 학습 구조 작성
```

## 이번 장의 과제

1. `(samples, 24, 2)` 모양을 자기 말로 설명해보자.
2. 입력은 PM10과 PM2.5 두 개지만 출력이 `Dense(1)`인 이유를 설명해보자.
3. PM2.5를 추가하면 항상 성능이 좋아진다고 말할 수 없는 이유를 설명해보자.
4. PM10 예측에 추가하면 좋을 다른 특성을 세 가지 적어보자.
