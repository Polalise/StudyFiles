# 12. 전이학습으로 꽃 분류 개선하기

## 이번 장의 목표

10장에서는 꽃 이미지를 분류하는 CNN을 처음부터 만들었다. 11장에서는 그 모델이 어떤 이미지를 맞히고 틀리는지 분석했다.

이번 장에서는 더 실전적인 방법을 배운다.

> 이미 많은 이미지를 보고 배운 모델을 가져와서, 꽃 분류 문제에 맞게 다시 사용하는 방법

이 방법을 전이학습(transfer learning)이라고 한다.

## 전이학습이란 무엇인가

전이학습은 쉽게 말해 “이미 배운 능력을 다른 문제에 옮겨 쓰는 것”이다.

사람으로 비유하면 이렇다.

```text
완전히 처음 보는 사람:
사진에서 선, 색, 모양, 물체가 무엇인지 처음부터 배워야 한다.

이미 사진을 많이 본 사람:
선, 색, 질감, 물체의 윤곽은 이미 잘 안다.
이제 꽃 이름만 새로 배우면 된다.
```

10장의 CNN은 첫 번째 경우에 가깝다. 꽃 데이터만 보고 처음부터 이미지 특징을 배운다.

전이학습은 두 번째 경우에 가깝다. 이미 대규모 이미지 데이터로 학습된 모델을 가져오고, 마지막 판단 부분만 꽃 분류에 맞게 바꾼다.

## 왜 전이학습이 유리할까

꽃 데이터는 약 3,670장이다. 처음 배우기에는 충분하지만, 큰 CNN을 안정적으로 학습시키기에는 많지 않을 수 있다.

이미지 모델은 초반에 이런 기본 특징을 배운다.

```text
선
모서리
색 변화
무늬
질감
둥근 형태
뾰족한 형태
```

이런 특징은 고양이, 자동차, 꽃, 음식 이미지에서 모두 어느 정도 공통으로 쓰인다.

그래서 이미 많은 이미지를 보고 이런 기본 특징을 배운 모델을 가져오면, 작은 데이터셋에서도 더 좋은 출발점에서 학습할 수 있다.

## 이번 장에서 사용할 모델: MobileNetV2

이번 장에서는 `MobileNetV2`를 사용한다.

MobileNetV2는 비교적 가볍게 설계된 이미지 모델이다. 아주 큰 모델보다 CPU 환경에서 다루기 쉽고, 전이학습 입문용으로 적당하다.

이번 장에서는 MobileNetV2 전체를 새로 학습하지 않는다.

대신 다음처럼 사용한다.

```text
입력 이미지
-> MobileNetV2가 이미지 특징 추출
-> 우리가 붙인 Dense 층이 꽃 종류 판단
```

즉, MobileNetV2는 “이미지를 보는 눈” 역할을 하고, 우리가 붙이는 마지막 층은 “꽃 이름을 고르는 판단기” 역할을 한다.

## include_top=False의 의미

Keras에서 사전학습 모델을 불러올 때 자주 보는 옵션이 있다.

```python
include_top=False
```

여기서 `top`은 원래 모델의 마지막 분류 부분을 뜻한다.

MobileNetV2는 원래 1,000개 종류의 이미지를 분류하도록 학습되어 있다. 하지만 우리는 1,000개가 아니라 꽃 5종을 분류하고 싶다.

그래서 원래 마지막 분류기는 떼어낸다.

```text
원래 MobileNetV2:
이미지 특징 추출 부분 + 1000종 분류기

우리가 원하는 구조:
이미지 특징 추출 부분 + 꽃 5종 분류기
```

이때 `include_top=False`를 사용한다.

## base_model.trainable = False의 의미

전이학습 초반에는 보통 사전학습 모델을 얼려 둔다.

```python
base_model.trainable = False
```

이 말은 “MobileNetV2가 이미 배운 부분은 당장 고치지 말자”는 뜻이다.

왜 얼릴까?

처음부터 모든 층을 고치면 작은 꽃 데이터에 맞춰 기존의 좋은 특징이 망가질 수 있다. 그래서 먼저 마지막 판단기만 학습한다.

비유하면 이렇다.

```text
이미 사진 보는 법을 잘 아는 선생님은 그대로 둔다.
대신 꽃 이름을 말하는 마지막 답안 작성법만 새로 배운다.
```

## fine-tuning은 무엇인가

전이학습에는 보통 두 단계가 있다.

```text
1단계: 특징 추출기 얼리기
2단계: 일부 층을 조금 풀어서 미세 조정하기
```

2단계를 fine-tuning이라고 한다.

하지만 fine-tuning은 처음부터 무리해서 하지 않아도 된다. 이번 장에서는 먼저 “얼린 사전학습 모델 + 새 분류기” 구조를 이해하는 데 집중한다.

fine-tuning은 다음 장 또는 프로젝트 개선 단계에서 다룰 수 있다.

## 이번 장의 실습 흐름

```text
꽃 이미지 데이터셋 준비
-> MobileNetV2 전처리 방식 적용
-> MobileNetV2 불러오기
-> 기존 특징 추출 부분 얼리기
-> 꽃 5종 분류기 붙이기
-> 학습
-> 저장
```

## 주의할 점

사전학습 모델을 처음 불러올 때 인터넷 연결이 필요할 수 있다. Keras가 MobileNetV2의 가중치 파일을 내려받아야 하기 때문이다.

한 번 내려받으면 보통 로컬 캐시에 저장되어 다시 사용할 수 있다.

현재 컴퓨터는 강한 딥러닝 GPU 환경이 아니므로, 이 장에서도 epoch는 작게 시작하는 것이 좋다.

## 포트폴리오에서의 의미

전이학습을 사용했다는 것은 단순히 모델을 가져다 썼다는 뜻이 아니다.

포트폴리오에서는 이렇게 설명할 수 있어야 한다.

```text
처음부터 작은 CNN을 학습한 뒤 오답 패턴을 분석했다.
데이터 수가 제한적이므로 사전학습 이미지 모델을 특징 추출기로 사용하는 전이학습을 적용했다.
MobileNetV2의 기존 특징 추출 층은 동결하고, 꽃 5종 분류기만 새로 학습했다.
이를 통해 작은 데이터셋에서도 더 안정적인 이미지 분류 모델을 만들고자 했다.
```

이 설명은 “딥러닝을 상황에 맞게 고를 수 있다”는 신호가 된다.

## 이번 장의 과제

1. 전이학습을 “이미 배운 능력을 옮겨 쓰는 것”이라는 말로 직접 설명해보자.
2. `include_top=False`가 왜 필요한지 설명해보자.
3. `base_model.trainable = False`가 무슨 뜻인지 설명해보자.
4. 처음부터 만든 CNN과 전이학습 모델의 장단점을 비교해보자.
